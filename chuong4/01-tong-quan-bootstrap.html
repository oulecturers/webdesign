4.1 · Tổng quan Bootstrap

Framework CSS phổ biến nhất: dựng giao diện đẹp, responsive rất nhanh nhờ hệ lưới và kho thành phần dựng sẵn.

1Bootstrap là gì?

Bootstrap là một framework CSS mã nguồn mở (Twitter, 2011) gồm sẵn: hệ lưới 12 cột responsive, hàng trăm lớp tiện ích (utility) và một kho thành phần (nút, card, navbar, modal, carousel...). Thay vì tự viết CSS từ đầu, ta ghép các class có sẵn để ra giao diện chuẩn, đồng nhất, chạy tốt trên mọi màn hình.

  • Mobile-first — thiết kế cho màn hẹp trước, mở rộng dần bằng breakpoint.
  • Phiên bản 5 (bài này dùng 5.3) — không cần jQuery, dùng JavaScript thuần + thư viện Popper cho tooltip/dropdown.
  • Tuỳ biến bằng biến CSS (--bs-*) và hỗ trợ chế độ sáng/tối qua data-bs-theme.
Vì sao học? Bootstrap giúp làm nhanh bản mẫu (prototype) và các trang quản trị. Sau khi nắm HTML/CSS thuần (Chương 2), bạn sẽ hiểu mỗi class Bootstrap thực chất làm gì.

2Cài đặt: CDN và npm

Cách 1 — CDN (nhanh nhất, không cần cài đặt): dán 2 dòng dưới. CSS đặt trong <head>; JavaScript đặt trước thẻ </body>. Bản bundle đã kèm sẵn Popper — cần cho Dropdown, Tooltip, Modal...

<!-- trong <head> -->
<link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css" rel="stylesheet">

<!-- ngay trước </body> -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

Cách 2 — npm (dự án thật, dùng cùng Vite/Webpack):

npm install bootstrap@5.3.3

// trong file JS đầu vào
import 'bootstrap/dist/css/bootstrap.min.css';
import 'bootstrap';   // kèm Popper
Lưu ý. Không nạp hai lần Bootstrap. Nếu dùng CDN thì thôi npm và ngược lại. Bản bootstrap.min.js (không có bundle) không kèm Popper — khi đó phải nạp Popper riêng.

3Trang Bootstrap đầu tiên

Chỉ cần thêm class, ta có nút, huy hiệu và hộp thông báo chuẩn — không viết dòng CSS nào:

Kết quả
Mới
Xin chào Bootstrap!
<button class="btn btn-primary">Nút chính</button>
<button class="btn btn-outline-secondary">Nút viền</button>
<span class="badge text-bg-success">Mới</span>
<div class="alert alert-success">Xin chào Bootstrap!</div>

4Utility-first hay Component?

Bootstrap cho cả hai cách dùng, và bạn thường trộn lẫn:

CáchVí dụKhi nào dùng
Component
khối dựng sẵn		 <div class="card">…</div> Cần nhanh một khối hoàn chỉnh (card, navbar, modal).
Utility
từng lớp nhỏ		 class="d-flex gap-2 p-3 rounded" Tinh chỉnh khoảng cách, màu, bố cục mà không viết CSS.
Đối chiếu Chương 5. Tailwind CSS đi theo hướng utility-first hoàn toàn (không có component dựng sẵn). Bootstrap thiên về component nhưng vẫn có nhiều utility. Bạn sẽ so sánh trực tiếp ở Chương 5.

5Thành phần chỉ-CSS và thành phần cần JS

Nhiều thứ chỉ cần CSS (nút, card, lưới, bảng). Nhưng các thành phần tương tác bắt buộc nạp bootstrap.bundle.min.js:

  • JS Dropdown, Collapse, Offcanvas
  • JS Modal, Carousel, ScrollSpy
  • JS Tooltip, Popover, Toast
  • CSS Nút, Card, Grid, Bảng, Badge
Lỗi thường gặp. Bấm nút mở Modal/Dropdown mà "không phản ứng" — gần như luôn do quên nạp file JS hoặc đặt sai vị trí. Hãy để thẻ <script> ngay trước </body>.
