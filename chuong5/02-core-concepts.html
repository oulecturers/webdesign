Core concepts — biến thể, dark mode, tuỳ biến

Những khái niệm lõi giúp bạn dùng Tailwind v4 thành thạo: giá trị tuỳ chỉnh, variant trạng thái, group/peer, dark mode và cách tuỳ biến bằng chỉ thị CSS-first.

1Giá trị tuỳ chỉnh & ba "lối thoát"

Luôn ưu tiên utility có sẵn (theo thang thiết kế). Khi cần một giá trị nằm ngoài thang, Tailwind v4 cho ba "lối thoát" (escape hatch):

  • Giá trị tuỳ ý [value] — vd top-[117px], bg-[#316ff6], grid-cols-[1fr_500px_2fr]. Dấu gạch dưới _ = khoảng trắng.
  • Thuộc tính không có utility [prop:value] — vd [mask-type:luminance].
  • Biến CSS (--var) = rút gọn của var(--var) — vd bg-(--brand). Có thể gợi ý kiểu: text-(length:--x).
Kết quả
Hộp rộng đúng w-[250px], nền bg-[#316ff6] — màu & kích thước nằm ngoài bảng mặc định. 
<!-- utility có sẵn dùng TRƯỚC; chỉ "thoát" khi cần -->
<!-- [value]: dấu _ = khoảng trắng -->
<div class="top-[117px] bg-[#316ff6] grid-cols-[1fr_500px_2fr]"></div>

<!-- [prop:value]: thuộc tính không có utility -->
<div class="[mask-type:luminance]"></div>

<!-- (--var): đọc biến CSS = var(--var); gợi ý kiểu bằng (length:--x) -->
<div class="bg-(--brand) text-(length:--x)"></div>

Mẹo

Chỉ dùng [ ] / ( ) khi giá trị thật sự nằm ngoài thang thiết kế. Lạm dụng "magic number" làm mất tính nhất quán — thứ chính là điểm mạnh của Tailwind.

2Biến thể trạng thái (variant)

Biến thể áp một utility chỉ trong một trạng thái, cú pháp variant:class. Có thể xếp chồng nhiều biến thể (đọc từ phải sang trái) và dùng biến thể tuỳ ý để nhắm phần tử con.

<!-- xếp chồng: chỉ áp khi dark + màn hình md + đang hover -->
<button class="dark:md:hover:bg-blue-700"></button>

<!-- biến thể tuỳ ý: nhắm mọi <p> con bằng & -->
<div class="[&_p]:mt-4"></div>

3Group & Peer

group + group-hover:*: con đổi kiểu theo trạng thái của CHA mang class group. peer + peer-checked:*: một phần tử đổi kiểu theo anh em đứng TRƯỚC mang class peer.

Kết quả

Di chuột vào cả hộp

Tiêu đề đổi màu theo trạng thái hover của CHA (.group).

Đã chọn!

<!-- group: con nghe theo CHA -->
<div class="group">
  <h1 class="group-hover:text-red-500">Tiêu đề</h1>
</div>

<!-- peer: nghe theo anh em ĐỨNG TRƯỚC -->
<input type="checkbox" class="peer">
<p class="hidden peer-checked:block">Đã chọn!</p>

4Variant hiện đại: has- / before-after / data-*

  • has-checked:* — dựa trên :has(), cho CHA đổi kiểu theo CON (ngược chiều với group).
  • before: / after: — tạo pseudo-element, bắt buộc có content-[...].
  • data-* — biến thể theo thuộc tính data-*, thường điều khiển bằng JS.
Kết quả — tích ô để cả thẻ đổi kiểu

Xem thêm

<!-- :has() — CHA đổi kiểu khi CON được tích -->
<label class="has-checked:bg-indigo-50 has-checked:ring-indigo-500">
  <input type="checkbox"> Ghi nhớ đăng nhập
</label>

<!-- before/after cần content-[...] -->
<a class="after:content-['_↗']">Xem thêm</a>

<!-- data-* điều khiển bằng JS: đổi thuộc tính là đổi kiểu -->
<div data-active="true" class="opacity-50 data-[active=true]:opacity-100"></div>

Ghi nhớ chiều tác động

group-*: con nghe CHA · peer-*: nghe anh em trước · has-*: CHA nghe CON. Ba biến thể phủ gần hết nhu cầu "đổi kiểu theo ngữ cảnh" mà không cần JS.

5Dark mode

Mỗi utility có một cặp dark: để khai báo kiểu cho chế độ tối, vd bg-white dark:bg-gray-900 text-gray-900 dark:text-white. Có 2 chiến lược:

  • Theo hệ điều hành (mặc định): dựa trên prefers-color-scheme — người dùng chỉnh sáng/tối ở OS.
  • Bật/tắt bằng tay: khai báo @custom-variant dark (&:where(.dark, .dark *)) rồi thêm class .dark lên một tổ tiên (thường là <html>).
Kết quả — thẻ dưới nằm trong <div class="dark">

Thẻ này đã áp kiểu tối

Nền tối, chữ sáng — các class dark: kích hoạt nhờ class .dark bọc ngoài (trang đã khai báo custom-variant sẵn).

/* v4: bật dark bằng class thay cho tuỳ chọn darkMode:'class' của v3 */
@custom-variant dark (&:where(.dark, .dark *));

<!-- mỗi utility có cặp dark: -->
<div class="bg-white dark:bg-gray-900 text-gray-900 dark:text-white"></div>

Lưu ý

Đặt đoạn JS đọc localStorage và gắn class .dark ngay trong <head> (trước khi body render) để tránh nháy sáng (FOUC). Tailwind v4 đã bỏ tuỳ chọn darkMode:'class' — thay bằng @custom-variant ở trên.

6Tuỳ biến nâng cao: @apply, @utility, !

  • @apply — nhồi nhiều utility vào MỘT class CSS (gom pattern lặp lại thành component).
  • @utility — tạo utility mới (thay cách viết plugin cũ của v3).
  • ! — đánh dấu !important, đặt ở CUỐI class: bg-red-500! (v3 dùng tiền tố !bg-red-500).
/* @apply — gom utility thành một class CSS */
.btn {
  @apply rounded-lg bg-sky-500 px-4 py-2 text-white;
}

/* @utility — tạo utility MỚI dùng được như class Tailwind */
@utility btn-lg {
  padding: 1rem 2rem;
  font-size: 1.25rem;
}
<!-- ! = !important, đặt CUỐI class (v3 là tiền tố !bg-red-500) -->
<div class="bg-red-500!"></div>

Chuẩn nghề

Dùng @apply có chừng mực — chỉ cho pattern thật sự lặp lại. Với dự án component (React/Vue) nên tách thành component thay vì gom class, giữ HTML là "nguồn sự thật" duy nhất về kiểu.

7Chỉ thị & theme variables

Tailwind v4 cấu hình bằng CSS. Dưới đây là các chỉ thị/hàm hay dùng:

Chỉ thị / hàmCông dụng
@import "tailwindcss";Nạp Tailwind — thay 3 lệnh @tailwind của v3.
@theme { … }Khai báo token thiết kế → tự sinh utility.
@applyDùng utility bên trong một class CSS thường.
@utilityĐịnh nghĩa utility mới (thay plugin cũ).
@variant / @custom-variantÁp / định nghĩa biến thể (vd dark).
--spacing(n) / --alpha(c / 50%)Hàm tính khoảng cách / pha độ mờ của màu.

Các nhóm theme variable chính (khai báo trong @theme, mỗi nhóm tự sinh utility tương ứng):

  • --color-* → bg-/text-/border-/ring-
  • --font-* / --text-* / --font-weight-*
  • --spacing — MỘT biến gốc cho p-/m-/w-/gap-
  • --radius-* / --shadow-*
  • --breakpoint-* / --container-*
  • --ease-* / --animate-*

Vì sao chỉ MỘT biến --spacing?

Từ một biến gốc, Tailwind suy ra toàn bộ thang p-*, m-*, w-*, gap-*… nên hệ khoảng cách luôn nhất quán. Đổi một biến là đổi cả hệ — đây là sức mạnh của cấu hình CSS-first trong v4.
