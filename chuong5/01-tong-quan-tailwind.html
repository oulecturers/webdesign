5.1 · Tổng quan Tailwind CSS

Triết lý utility-first: dựng giao diện bằng những "viên gạch" class nhỏ ngay trong HTML — mỗi class là một dòng CSS đã học ở Chương 2.

1Utility-first là gì?

Thay vì viết một file CSS riêng, Tailwind cho hàng nghìn class tiện ích (utility) — mỗi class ứng với một quy tắc CSS. Ta ghép các "viên gạch nhỏ" ngay trong HTML để tự dựng thành phần theo ý muốn.

Kết quả
<!-- ghép các "viên gạch nhỏ" để tự dựng nút -->
<button class="bg-blue-500 hover:bg-blue-700
               text-white font-bold py-2 px-4 rounded">
  Click here!
</button>

Phân rã class

bg-blue-500 nền xanh · hover:bg-blue-700 đậm khi hover · text-white font-bold · py-2 px-4 padding · rounded bo góc.

2Ưu & khuyết điểm

Ưu điểm

  • Phát triển nhanh — không viết CSS thủ công
  • Dễ tuỳ chỉnh qua @theme (biến CSS)
  • Responsive & dark mode tích hợp sẵn
  • Tái sử dụng linh hoạt, hỗ trợ trạng thái (hover/focus/active)

Khuyết điểm

  • HTML nhiều class → dài, khó đọc ban đầu
  • Cần thời gian nhớ tên class
  • Khắc phục: Tailwind IntelliSense (VS Code)
  • và @apply / tách component (React/Vue)

3Tailwind vs Bootstrap

Tiêu chíTailwind (utility-first)Bootstrap (component-based)
Cách dùngGhép class nhỏ tự dựng UIComponent sẵn (button, card, modal)
Tuỳ chỉnhRất linh hoạt (nhiều class)Dễ dùng nhưng ít linh hoạt
Kích thướcNhẹ (chỉ giữ class dùng tới)Lớn (nhiều component mặc định)

Vì sao học CSS thuần trước rồi mới Tailwind?

Mỗi class Tailwind chỉ là một dòng CSS đã học ở Chương 2 (flex = display:flex, pt-4 = padding-top:1rem). Hiểu CSS thì học Tailwind rất nhanh và biết class nào sinh ra CSS gì.

4Cài đặt: CDN (học) vs npm (production)

Giáo trình dùng Tailwind v4 (v4.3, 2026) — v4 đổi cách cài đặt & cấu hình so với v3.

<!-- Học nhanh: Play CDN trình duyệt (1 dòng, không cài) -->
<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>

# Dự án thật: cài qua npm (cú pháp Tailwind v4)
npm install tailwindcss @tailwindcss/cli
npx @tailwindcss/cli -i input.css -o output.css --watch

/* input.css — v4 chỉ cần MỘT dòng (thay 3 chỉ thị @tailwind của v3) */
@import "tailwindcss";

Chuẩn nghề

Production không dùng Play CDN (tải cả Tailwind, chậm). Cài qua npm — v4 tự dò file để chỉ giữ class thực dùng (automatic content detection, không cần khai báo content/purge như v3) → CSS cuối chỉ vài KB.

5Tuỳ biến theme — v4 cấu hình bằng CSS

v3 dùng tailwind.config.js (JS); v4 khai báo biến thương hiệu ngay trong CSS bằng khối @theme. Trang này đã khai báo --color-brand: #04AA6D → sinh ra class bg-brand, text-brand…

Kết quả — class thương hiệu tự sinh
bg-brand text-brand border-brand 
/* khối @theme tạo ra các class thương hiệu */
@import "tailwindcss";
@theme {
  --color-brand: #04AA6D;     /* → class bg-brand, text-brand… */
  --font-display: "Poppins", sans-serif;
}

Thực tế doanh nghiệp

Tailwind là cách viết CSS phổ biến nhất cho dự án mới, thường đi kèm React/Vue + Vite (v4 có plugin Vite chính thức) — điểm cộng khi xin việc Front-end.
