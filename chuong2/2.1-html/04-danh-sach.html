Danh sách trong HTML — ul · ol · dl

Danh sách giúp nhóm các mục liên quan thành khối có cấu trúc: danh sách không thứ tự, có thứ tự và danh sách mô tả. Đi từ cách dùng cơ bản đến tuỳ biến bằng CSS và biến danh sách thành menu điều hướng.

1 Danh sách không thứ tự <ul> Cơ bản

Dùng khi thứ tự các mục không quan trọng. Mỗi mục là một thẻ <li> (list item) đặt bên trong <ul> (unordered list). Mặc định trình duyệt hiển thị một chấm tròn đặc (●) trước mỗi mục.

Kết quả
  • Cà phê
  • Trà
  • Sữa

Chỉ được đặt <li> làm con trực tiếp của <ul>. Bên trong mỗi <li> thì lại có thể chứa văn bản, ảnh, liên kết, thậm chí cả một danh sách con (xem mục 5).

Cú pháp:

<ul>
  <li>Cà phê</li>
  <li>Trà</li>
  <li>Sữa</li>
</ul>

<ul> và <li> đều là phần tử khối (block), nên mỗi mục tự xuống một dòng riêng.

2 Kiểu dấu đầu dòng — list-style-type Cơ bản

Thuộc tính CSS list-style-type đổi hình dạng dấu đầu dòng của <ul>. Bốn giá trị hay dùng: disc (chấm đặc — mặc định), circle (vòng tròn rỗng), square (vuông đặc) và none (không có dấu).

Kết quả · 4 kiểu cạnh nhau

disc (mặc định)

  • Mục một
  • Mục hai

circle

  • Mục một
  • Mục hai

square

  • Mục một
  • Mục hai

none

  • Mục một
  • Mục hai
/* đặt kiểu dấu đầu dòng cho ul */
ul.disc   { list-style-type: disc; }   /* ● mặc định */
ul.circle { list-style-type: circle; } /* ○ */
ul.square { list-style-type: square; } /* ▪ */
ul.none   { list-style-type: none; }   /* (bỏ dấu) */

list-style-type:none rất hay dùng để "xoá" dấu đầu dòng khi biến danh sách thành thanh menu hoặc danh sách thẻ — sẽ gặp lại ở mục 7.

3 Danh sách có thứ tự <ol> Cơ bản

Dùng khi thứ tự quan trọng (các bước, bảng xếp hạng…). Mặc định đánh số 1, 2, 3… Thuộc tính HTML type đổi kiểu đánh số; start đặt số bắt đầu; reversed đảo ngược.

3.1 — Mặc định & thuộc tính type

Kết quả · các kiểu type: 1 · A · a · I · i

type="1"

  1. Một
  2. Hai
  3. Ba

type="A"

  1. Một
  2. Hai
  3. Ba

type="a"

  1. Một
  2. Hai
  3. Ba

type="I"

  1. Một
  2. Hai
  3. Ba

type="i"

  1. Một
  2. Hai
  3. Ba

3.2 — Thuộc tính start và reversed

start="5" — bắt đầu từ 5
  1. Mục thứ năm
  2. Mục thứ sáu
  3. Mục thứ bảy
reversed — đếm ngược 3, 2, 1
  1. Hạng ba
  2. Hạng nhì
  3. Hạng nhất
/* type, start, reversed là thuộc tính HTML (không phải CSS) */
<ol type="A">            <!-- đánh số A, B, C -->
<ol start="5">          <!-- bắt đầu từ 5 -->
<ol reversed>            <!-- đếm ngược -->
  <li value="10">...</li> <!-- value: đặt số riêng cho 1 mục -->
</ol>

Có thể đặt giá trị riêng cho một mục bằng thuộc tính value trên <li>; các mục sau sẽ tiếp tục đếm từ giá trị đó.

4 Danh sách mô tả <dl> Cơ bản

Danh sách mô tả (description list) ghép từng thuật ngữ với định nghĩa của nó — như một cuốn từ điển hay bảng chú giải. Gồm ba thẻ: <dl> (bao ngoài), <dt> (thuật ngữ) và <dd> (mô tả).

Kết quả · bảng chú giải
HTML
Ngôn ngữ đánh dấu siêu văn bản — tạo cấu trúc nội dung trang web.
CSS
Ngôn ngữ định kiểu — quy định màu sắc, phông chữ, bố cục.
JavaScript
Ngôn ngữ lập trình — thêm tính tương tác cho trang web.
<dl>
  <dt>HTML</dt>
  <dd>Ngôn ngữ đánh dấu siêu văn bản…</dd>
  <dt>CSS</dt>
  <dd>Ngôn ngữ định kiểu…</dd>
</dl>

Một <dt> có thể đi kèm nhiều <dd> (một thuật ngữ, nhiều nghĩa), và ngược lại nhiều <dt> có thể chung một <dd> (nhiều từ đồng nghĩa).

5 Danh sách lồng nhau (nested) Cơ bản

Có thể đặt một danh sách bên trong một <li> để tạo cấu trúc nhiều cấp: menu nhiều tầng, cây thư mục, mục lục… Lưu ý <ul>/<ol> con phải nằm trong thẻ <li>, không nằm trực tiếp dưới <ul> cha.

Menu nhiều cấp
  • Đồ uống
    • Cà phê
      • Đen
      • Sữa
    • Trà
  • Đồ ăn
    • Bánh ngọt
    • Snack
Cây thư mục (dấu tuỳ biến)
  • du-an
    • index.html
    • css
      • style.css
<ul>
  <li>Đồ uống
    <ul>                 <!-- danh sách con nằm TRONG <li> -->
      <li>Cà phê</li>
      <li>Trà</li>
    </ul>
  </li>                      <!-- đóng <li> cha SAU danh sách con -->
</ul>

Lỗi thường gặp: đóng </li> cha trước khi mở danh sách con, khiến cây bị "vỡ" thụt lề. Hãy đóng </li> cha sau khi đã đóng danh sách con.

6 Tuỳ biến list bằng CSS — list-style Nâng cao

Thuộc tính rút gọn list-style gộp ba thuộc tính: list-style-type (kiểu dấu), list-style-position (vị trí inside/outside) và list-style-image (ảnh làm dấu).

6.1 — Vị trí dấu: inside vs outside

outside (mặc định) — dấu nằm NGOÀI lề chữ
  • Khi xuống dòng, chữ canh thẳng và dấu nhô ra ngoài.
  • Mục hai.
inside — dấu nằm TRONG, chữ bao quanh dấu
  • Khi xuống dòng, chữ thụt theo dấu nằm bên trong khối.
  • Mục hai.

6.2 — Dùng ảnh / counter làm dấu đầu dòng

Ảnh SVG làm dấu (background-image)
  • Hoàn thành chương 1
  • Hoàn thành chương 2
  • Ôn tập cuối kỳ
Đánh số tròn bằng CSS counter
  1. Tạo file HTML
  2. Liên kết CSS
  3. Kiểm tra trên trình duyệt
/* list-style rút gọn: type | position | image */
ul { list-style: square inside; }

/* dùng ảnh làm dấu (hoặc background-image cho linh hoạt hơn) */
ul { list-style-image: url(check.svg); }

/* đánh số tuỳ biến bằng counter */
ol { list-style: none; counter-reset: muc; }
ol li { counter-increment: muc; }
ol li::before { content: counter(muc); }

Dùng background-image + padding-left (như demo trên) thường dễ căn chỉnh kích thước/vị trí dấu hơn list-style-image.

7 Biến danh sách thành menu ngang Nâng cao

Bài thực hành kinh điển của W3Schools: thanh điều hướng (navbar) bản chất là một <ul>. Ta bỏ dấu đầu dòng, xoá lề mặc định rồi xếp các <li> nằm ngang.

7.1 — Cách cổ điển: li { display: inline-block }

Kết quả · navbar bằng inline-block
  • Trang chủ
  • Giới thiệu
  • Dịch vụ
  • Liên hệ
ul { list-style-type: none;   /* bỏ dấu đầu dòng */
       margin: 0; padding: 0;  /* xoá lề/khoảng đệm mặc định */
       overflow: hidden;
       background: #16407a; }
li    { display: inline-block; }  /* li vốn block → ép thành 1 hàng */
li a  { display: block; color: #fff;
        padding: 12px 18px; text-decoration: none; }
li a:hover { background: #1D559F; }

7.2 — Cách hiện đại: display: flex

Kết quả · navbar bằng Flexbox
  • Trang chủ
  • Sản phẩm
  • Tin tức
  • Liên hệ
ul { list-style: none; margin: 0; padding: 0;
       display: flex; gap: 4px; }  /* xếp ngang, có khoảng cách */
li a { display: block; padding: 12px 18px;
        color: #fff; text-decoration: none; }

Đặt <a> thành display:block giúp cả ô (kể cả phần đệm) đều bấm được, không chỉ riêng phần chữ. Ngày nay Flexbox được ưu tiên vì dễ căn giữa và chia khoảng cách.

9 Khi nào dùng ul / ol / dl? Tổng kết

Dùng <ul> khi thứ tự không quan trọng: danh sách tính năng, mục giỏ hàng, các liên kết menu. Đây cũng là nền tảng cho mọi navbar.

Dùng <ol> khi thứ tự có ý nghĩa: các bước hướng dẫn, bảng xếp hạng, danh sách đánh số tham chiếu.

Dùng <dl> cho cặp thuật ngữ – định nghĩa: bảng chú giải, FAQ (câu hỏi/đáp), thông số kỹ thuật, metadata (tên : giá trị).

Mẹo navbar. Thanh điều hướng nên dựng từ <ul> chứa các <li> > <a>, bọc trong <nav>. Cấu trúc danh sách giúp trình đọc màn hình thông báo "menu có N mục" — tốt cho khả năng truy cập (accessibility).

Tránh. Đừng dùng danh sách chỉ để thụt lề văn bản, và đừng nhồi nội dung không phải mục con (vd <p> "lạc lõng") trực tiếp vào <ul>/<ol> — con trực tiếp hợp lệ chỉ là <li>.
