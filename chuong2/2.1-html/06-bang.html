1 Bảng cơ bản: table · tr · td · th Cơ bản

Một bảng gồm các hàng <tr> (table row), trong mỗi hàng có các ô dữ liệu <td> (table data). Ô tiêu đề dùng <th> (table header) — chữ in đậm & canh giữa mặc định, có ý nghĩa ngữ nghĩa cho trình đọc màn hình.

Kết quả
STTHọ tênMSSV
1Dương Hải Đăng0712031
2Dương Kiều Lam0712036
3Lê Ngọc Trân0712044

Thứ tự ghi nhớ: <table> → bên trong là các <tr> (hàng) → bên trong mỗi hàng là các <th>/<td> (ô). Số ô của các hàng nên bằng nhau.

<th> dùng cho ô tiêu đề (đầu cột hoặc đầu hàng), <td> cho ô dữ liệu thường.

<table>
  <tr>                              <!-- hàng tiêu đề -->
    <th>STT</th> <th>Họ tên</th> <th>MSSV</th>
  </tr>
  <tr>                              <!-- một hàng dữ liệu -->
    <td>1</td> <td>Dương Hải Đăng</td> <td>0712031</td>
  </tr>
</table>

Bảng "trần" không có viền vì viền do CSS tạo ra. Thuộc tính HTML cũ border="1" hay cellpadding đã lỗi thời — ở mục 3 ta sẽ thêm viền đúng cách bằng CSS.

2 Tiêu đề bảng <caption> Cơ bản

<caption> đặt một tên/chú thích cho cả bảng. Nó phải là phần tử đầu tiên ngay sau <table>. Caption tốt cho khả năng truy cập (a11y) vì giúp người dùng hiểu bảng nói về gì.

Kết quả · caption ở phía trên bảng
Bảng 1 — Điểm thi cuối kỳ lớp Thiết kế Web
Họ tênĐiểm
Nguyễn Đình Huy9.0
Trần Thị Mai8.5
<table>
  <caption>Bảng 1 — Điểm thi cuối kỳ</caption>   <!-- phải đứng ĐẦU TIÊN -->
  <tr>...</tr>
</table>

/* CSS: đổi vị trí caption */
table caption { caption-side: bottom; }   /* top (mặc định) | bottom */

Dùng CSS caption-side: bottom để đưa chú thích xuống dưới bảng, và text-align để canh trái/giữa/phải theo ý.

3 Viền & gộp viền: border · border-collapse Cơ bản

Mặc định mỗi ô và cả bảng có viền riêng nên xuất hiện viền đôi và có khe hở giữa các ô. Thuộc tính border-collapse quyết định các viền đó tách rời hay gộp làm một.

separate (mặc định) — viền đôi, có khe
AB
12

border-collapse: separate; border-spacing: 4px;

collapse — gộp thành 1 đường mảnh
AB
12

border-collapse: collapse;

/* Cần ĐẶT VIỀN cho cả TABLE và từng Ô (th, td) */
table { border-collapse: collapse; }   /* gộp viền đôi → 1 đường */
th, td { border: 1px solid #b9c6da; }

/* Nếu để 'separate', điều khiển khe hở bằng: */
table { border-collapse: separate; border-spacing: 4px; }

Đa số bảng dữ liệu nên dùng border-collapse: collapse cho gọn, sạch. Chỉ đặt border lên <table> thôi chưa đủ — phải đặt cả cho th, td mới có lưới ô.

4 Khoảng đệm & canh chữ trong ô Cơ bản

Trong mỗi ô, dùng padding để tạo khoảng đệm, text-align để canh ngang (left/center/right) và vertical-align để canh dọc (top/middle/bottom).

Kết quả · các ô cố ý cao 64px để thấy rõ canh dọc
Canh tráiCanh giữaCanh phải
left · top center · middle right · bottom 
th, td { padding: 10px 14px; }       /* khoảng đệm trong ô */

td.gia    { text-align: right; }      /* số/tiền canh phải cho dễ đọc */
td.tieude { text-align: center; }
td.ghichu { vertical-align: top; }    /* canh đỉnh khi ô cao */

Quy ước trình bày dữ liệu: chữ canh trái, số/tiền/ngày canh phải, tiêu đề cột canh giữa. Điều này giúp mắt dò cột số liệu nhanh hơn.

5 Gộp ô: colspan & rowspan Cơ bản

Một ô có thể trải ra nhiều cột hoặc nhiều hàng: colspan="n" gộp n cột (mở rộng theo chiều ngang), rowspan="n" gộp n hàng (mở rộng theo chiều dọc).

Kết quả · "Họ tên" gộp 2 hàng, "Điểm" gộp 3 cột
Họ tên Điểm các môn
Môn 1Môn 2Môn 3
Dương Hữu Thành 9.09.510
Trần Thị Mai 8.09.58.5 
<tr>
  <th rowspan="2">Họ tên</th>   <!-- ô này cao bằng 2 hàng -->
  <th colspan="3">Điểm các môn</th>  <!-- rộng bằng 3 cột -->
</tr>
<tr>                              <!-- hàng này chỉ còn 3 ô (vì cột Họ tên đã bị chiếm) -->
  <th>Môn 1</th> <th>Môn 2</th> <th>Môn 3</th>
</tr>

Lưu ý đếm ô: khi một ô đã colspan/rowspan, các hàng/cột bị nó chiếm chỗ sẽ bớt đi số <td> tương ứng. Đếm sai là nguyên nhân số 1 khiến bảng "vỡ" lệch.

6 Nhóm bảng: <thead> · <tbody> · <tfoot> Cơ bản

Ba thẻ này chia bảng thành đầu bảng (tiêu đề), thân bảng (dữ liệu) và chân bảng (tổng kết). Chúng giúp code rõ ràng, dễ style theo từng vùng và hỗ trợ trình duyệt lặp lại tiêu đề khi in nhiều trang.

Kết quả · chân bảng tô vàng để tổng kết
Sản phẩmSố lượngThành tiền
Bàn phím2700.000đ
Chuột3450.000đ
Tai nghe1500.000đ
Tổng cộng1.650.000đ
 
<table>
  <thead> <!-- hàng tiêu đề -->
    <tr><th>Sản phẩm</th><th>SL</th><th>Thành tiền</th></tr>
  </thead>
  <tbody> <!-- dữ liệu chính -->
    <tr><td>Bàn phím</td><td>2</td><td>700.000đ</td></tr>
  </tbody>
  <tfoot> <!-- tổng kết / chân bảng -->
    <tr><td colspan="2">Tổng cộng</td><td>1.650.000đ</td></tr>
  </tfoot>
</table>

Bạn cứ viết theo thứ tự tự nhiên thead → tbody → tfoot; trình duyệt vẫn luôn hiển thị tfoot ở cuối. Mỗi bảng nên có 1 thead & 1 tfoot, nhưng có thể có nhiều tbody.

7 Nhóm cột: <colgroup> · <col> Nâng cao

Muốn định dạng cả một cột (vd tô màu cột giá) mà không phải sửa từng ô? Dùng <colgroup> chứa các <col> đặt ngay sau <caption> (trước <thead>).

Kết quả · cột "Mã" và cột "Giá" được tô riêng
MãSản phẩmGiá
SP01Bàn phím cơ350.000đ
SP02Chuột không dây150.000đ
SP03Tai nghe500.000đ
 
<table>
  <colgroup>
    <col class="col-id">     <!-- cột 1 -->
    <col>                  <!-- cột 2: để mặc định -->
    <col class="col-price">  <!-- cột 3 -->
  </colgroup>
  ...
</table>

.col-id    { background: #eaf1fb; }
.col-price { background: #fef3c7; }

<col> chỉ nhận được một vài thuộc tính CSS: background, width, border, visibility. Các thuộc tính khác (như color chữ, text-align) không áp được qua <col> — phải style trực tiếp ở td.

8 Tô màu xen kẽ (zebra) & hiệu ứng hover Nâng cao

Bảng nhiều hàng dễ đọc hơn khi tô màu xen kẽ — gọi là "zebra striping". Dùng bộ chọn tr:nth-child(even) và thêm tr:hover để làm nổi hàng đang trỏ tới.

Kết quả · rê chuột lên từng hàng để thấy hover
STTQuốc giaThủ đôDân số (triệu)
1Việt NamHà Nội99
2Nhật BảnTokyo125
3Hàn QuốcSeoul52
4Thái LanBangkok70
5PhápParis68
 
/* Tô màu các hàng CHẴN trong thân bảng */
tbody tr:nth-child(even) { background: #eef6ef; }

/* Làm sáng hàng khi rê chuột */
tbody tr:hover { background: #d1fae5; }

Dùng :nth-child(odd) để tô hàng lẻ thay vì chẵn. Chỉ nhắm vào tbody tr để không tô nhầm hàng tiêu đề ở thead.

9 Bề rộng bảng/ô & bảng cuộn ngang Nâng cao

Đặt bề rộng bằng CSS width trên table hoặc từng cột. Khi bảng có nhiều cột và màn hình hẹp, bọc bảng trong một <div> có overflow-x: auto để cuộn ngang thay vì vỡ trang.

Bảng có bề rộng cố định 320px:

Khoá họcTín chỉ
Thiết kế Web3
Cơ sở dữ liệu4
/* Bọc bảng để cuộn ngang trên màn hình hẹp */
.bao-bang { overflow-x: auto; }
.bao-bang table { min-width: 760px; }   /* không co nhỏ hơn 760px */

/* Chỉ định bề rộng từng cột */
table  { width: 100%; }
.id    { width: 60px; }
td     { white-space: nowrap; }   /* không xuống dòng giữa ô */

Mẹo "responsive table" phổ biến: bọc <table> trong <div style="overflow-x:auto">. Đơn giản, không cần thư viện, hoạt động trên mọi trình duyệt.

11 Đừng dùng bảng để dàn layout Lưu ý quan trọng

Đây là quy tắc bắt buộc nhớ: <table> chỉ dùng để trình bày DỮ LIỆU dạng bảng (bảng điểm, bảng giá, thống kê…). Tuyệt đối không dùng bảng để chia cột, đặt menu, dàn bố cục trang.

KHÔNG dùng <table> để dàn bố cục (header, sidebar, lưới ảnh…). Cách làm cũ này khiến HTML rối, khó bảo trì, kém truy cập (a11y) và không responsive.

NÊN dùng CSS Flexbox (dãy 1 chiều) và CSS Grid (lưới 2 chiều) để dàn trang. Học chi tiết ở mục 2.2 CSS & 2.3 Responsive.

Phân biệt nhanh:

• Đang trình bày dữ liệu có hàng & cột rõ ràng (mỗi ô là một mẩu dữ liệu)? → dùng <table>.

• Đang sắp đặt vị trí các khối giao diện trên trang? → dùng display: flex hoặc display: grid.
