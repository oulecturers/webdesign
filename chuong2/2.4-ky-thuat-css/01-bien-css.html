Biến CSS — var() & Custom Properties

1 Biến CSS là gì & lợi ích Cơ bản

Biến CSS (Custom Properties) là nơi lưu một giá trị có thể tái sử dụng. Thay vì gõ #1D559F ở 20 chỗ, ta đặt tên --primary một lần rồi dùng lại. Muốn đổi màu cả trang? Sửa đúng một dòng.

Kết quả · cả hai nút lấy màu từ cùng một biến
Thẻ này cũng dùng var(--d-primary) cho viền và var(--d-radius) cho bo góc. Cùng nguồn — thống nhất giao diện.

Cú pháp: khai báo & sử dụng

/* 1) KHAI BÁO biến: tên BẮT BUỘC bắt đầu bằng hai gạch -- */
:root {
  --d-primary: #1D559F;   /* tên biến phân biệt hoa/thường */
  --d-radius:  10px;
}

/* 2) SỬ DỤNG biến bằng hàm var() */
.btn {
  background: var(--d-primary);
  border-radius: var(--d-radius);
}

Vì sao nên dùng?

• Đổi một chỗ, áp mọi nơi — bảo trì cực nhanh.
• Dễ đọc — var(--primary) rõ nghĩa hơn một mã màu khó nhớ.
• Động — biến CSS sống trong DOM, có thể đổi theo nhánh hoặc bằng JavaScript (khác hẳn biến của Sass chỉ tồn tại lúc biên dịch).

Tên biến phân biệt chữ hoa/thường: --primary và --Primary là hai biến khác nhau. Quy ước phổ biến: viết thường, nối bằng gạch ngang (kebab-case).

2 Khai báo ở :root — phạm vi toàn cục Cơ bản

:root là bộ chọn trỏ tới phần tử gốc <html>. Biến đặt ở đây có phạm vi toàn cục: mọi phần tử trong trang đều dùng được.

Kết quả · bốn ô màu lấy từ biến toàn cục
--d-primary
--d-primary-d
--d-accent
bo góc --d-radius = 10px 
/* Gom toàn bộ "design token" của trang vào :root */
:root {
  --d-primary:   #1D559F;
  --d-primary-d: #16407a;
  --d-accent:    #e84393;
  --d-gap:       10px;
  --d-radius:    10px;
}

Đặt biến chung ở :root thay vì rải rác. Muốn đổi tông màu thương hiệu cả trang, bạn chỉ cần sửa vài dòng trong :root là xong.

3 Phạm vi (scope) — biến cục bộ Cơ bản

Biến không bắt buộc phải ở :root. Khai báo trong một bộ chọn bất kỳ thì biến đó chỉ sống bên trong phần tử đó và con cháu của nó. Nhờ vậy mỗi nhánh DOM có thể có giá trị riêng.

Kết quả · cùng một class .d-tag nhưng màu khác nhau theo nhánh
Nhánh A — --d-local: #04AA6D: Thẻ trong A Thẻ trong A
Nhánh B — --d-local: #6c5ce7: Thẻ trong B
Ngoài cả hai nhánh — --d-local chưa được đặt, dùng giá trị dự phòng xám: Thẻ "mồ côi" 
/* Cùng một class, màu lấy từ biến local của nhánh cha */
.d-tag   { background: var(--d-local, #94a3b8); } /* dự phòng xám */
.d-box-a { --d-local: #04AA6D; }  /* chỉ áp cho con của A */
.d-box-b { --d-local: #6c5ce7; }  /* chỉ áp cho con của B */

Biến cục bộ rất hợp với component: mỗi thẻ card/nút tự định nghĩa biến riêng (vd --card-pad), không lo ảnh hưởng phần khác của trang.

4 Giá trị dự phòng var(--x, mặc-định) Cơ bản

Hàm var() nhận tham số thứ hai làm giá trị dự phòng: nếu biến chưa được khai báo (hoặc không hợp lệ), trình duyệt dùng giá trị mặc định này thay vì bỏ qua.

Kết quả · biến không tồn tại → dùng dự phòng
var(--d-khong-ton-tai, #E63946) → đỏ var(--d-primary, #999) → xanh (biến có thật) 
/* Tham số sau dấu phẩy = giá trị dùng KHI biến chưa có */
.a { background: var(--chua-khai-bao, #E63946); } /* → đỏ dự phòng */
.b { background: var(--d-primary, #999); }       /* → xanh: biến có thật */

/* Dự phòng có thể lồng nhau */
.c { color: var(--x, var(--y, black)); }

Luôn cân nhắc giá trị dự phòng cho các biến "có thể chưa được set" — giúp giao diện không bị vỡ khi thiếu biến (ví dụ component dùng lại ở nơi chưa định nghĩa theme).

5 Kế thừa & cascade của biến Cơ bản

Biến CSS được kế thừa: phần tử con tự động nhận biến của cha. Nếu con khai báo lại cùng tên, giá trị mới ghi đè cho nhánh con đó (theo cơ chế cascade từ trên xuống).

Kết quả · con kế thừa --d-color, nhánh con ghi đè sang đỏ

Đoạn này kế thừa --d-color = #1D559F (xanh) từ cha.

Đoạn này cũng vậy — vẫn xanh.

Nhánh con đặt lại --d-color = #E63946 → đoạn này thành đỏ.

.d-inherit         { --d-color: #1D559F; }   /* cha đặt biến */
.d-inherit p       { color: var(--d-color); }  /* mọi p kế thừa */
.d-inherit .d-child{ --d-color: #E63946; }   /* nhánh con ghi đè */

Đây là điểm mạnh nhất của biến CSS so với biến Sass: chúng động theo DOM. Cùng một component, đặt trong nhánh có biến khác nhau sẽ tự đổi diện mạo — nền tảng của theme & dark mode.

6 Theme — đổi bảng màu đồng loạt Nâng cao

Khi nhiều phần tử cùng đọc một biến, chỉ cần đổi biến đó là cả khung đổi theo. Mỗi "theme" chỉ là một bộ giá trị biến khác nhau — không phải sửa từng phần tử.

Kết quả · 4 khung giống hệt, chỉ khác giá trị --d-t-primary

Theme mặc định

--d-t-primary: #1D559F

liên kết

Theme xanh lá

--d-t-primary: #04AA6D

liên kết

Theme tím

--d-t-primary: #6c5ce7

liên kết

Theme vàng

--d-t-primary: #d97706

liên kết 
/* Khung theme: tiêu đề, nút, thanh, link đều đọc --d-t-primary */
.d-theme        { --d-t-primary: #1D559F; }
.d-th-title     { color: var(--d-t-primary); }
.d-th-btn       { background: var(--d-t-primary); }

/* Đổi theme = đổi DUY NHẤT một biến */
.d-theme.green  { --d-t-primary: #04AA6D; }
.d-theme.purple { --d-t-primary: #6c5ce7; }

Mô hình "đổi biến để đổi theme" giúp tách giá trị khỏi cách dùng. Thêm theme mới chỉ là thêm một bộ biến, không phải viết lại CSS của từng thành phần.

7 Dark mode — đổi nhóm biến Nâng cao

Dark mode chỉ là một theme đặc biệt: đổi cả nhóm biến (nền, chữ, màu chủ đạo) sang tông tối. Ở đây ta đổi biến chỉ trong khung .d-theme.dark — không động tới :root nên khung bài học vẫn sáng bình thường.

.d-theme {
  --d-t-bg:      #ffffff;
  --d-t-text:    #1a1f29;
  --d-t-primary: #1D559F;
  background: var(--d-t-bg); color: var(--d-t-text);
}

/* Dark mode: chỉ ĐỔI NHÓM BIẾN — không sửa từng phần tử */

  --d-t-bg:      #0f172a;
  --d-t-text:    #e6edf3;
  --d-t-primary: #74b9ff;
}

/* Toggle thuần CSS: checkbox tích → ánh xạ sang bộ biến tối */
.d-darkbox:checked ~ .d-theme { --d-t-bg: #0f172a; /* ... */ }

Trong dự án thật thường dùng một trong hai cách:

• Theo cài đặt hệ điều hành: @media (prefers-color-scheme: dark) { :root { ... } }.
• Cho người dùng tự bật: JavaScript thêm class data-theme="dark" (hoặc .dark) lên <html>, rồi CSS đổi biến theo html.dark { --bg: ...; }.

Trong bài này dark mode cố tình bị giới hạn trong khung .d-theme để không phá giao diện trang học. Khi tự làm, bạn mới đổi biến ở :root/html cho toàn trang.

8 Biến + calc() Nâng cao

Có thể tính toán trên biến bằng calc(). Rất hợp để xây thang spacing từ một đơn vị gốc: padding = bội số của một biến gap chung.

Kết quả · padding = bội số của --d-gap (10px)
padding: var(--d-gap) → 10px
padding: calc(var(--d-gap) * 2) → 20px
padding: calc(var(--d-gap) * 3) → 30px
gap =
calc(
--d-gap
×1.5)
:root { --d-gap: 10px; }

.p1 { padding: var(--d-gap); }              /* 10px */
.p2 { padding: calc(var(--d-gap) * 2); }    /* 20px */
.p3 { padding: calc(var(--d-gap) * 3); }    /* 30px */

/* Lưu ý: bên trong calc() phải có khoảng trắng quanh dấu * và / */
.box { width: calc(100% - var(--d-gap) * 2); }

Trong calc() phải có khoảng trắng quanh các phép + - * / (vd calc(var(--g) * 2), không phải calc(var(--g)*2)). Đơn vị cũng phải hợp lệ khi cộng/trừ.

9 Đổi biến bằng JavaScript Nâng cao

Vì biến CSS sống trong DOM, JavaScript có thể đọc bằng getComputedStyle() và ghi bằng style.setProperty() — giao diện đổi ngay tức thì.

Kết quả · kéo thanh để đổi biến --d-js-h (màu HSL)
Màu nền lấy từ var(--d-js-h)

JS gọi setProperty('--d-js-h', ...) → CSS hsl(var(--d-js-h),70%,50%) cập nhật theo.

// ĐỌC giá trị biến (lưu ý cần .trim())
const root = getComputedStyle(document.documentElement);
const mau  = root.getPropertyValue('--d-primary').trim();

// GHI / đổi giá trị biến trên một phần tử
element.style.setProperty('--d-js-h', '120');

// Ví dụ thanh trượt đổi màu theo thời gian thực
range.addEventListener('input', () => {
  box.style.setProperty('--d-js-h', range.value);
});

getPropertyValue() trả về chuỗi, thường có khoảng trắng đầu — nhớ .trim(). Đặt biến bằng setProperty trên document.documentElement (tức :root) sẽ áp cho toàn trang.

Biến CSS được mọi trình duyệt hiện đại hỗ trợ. Với trình duyệt rất cũ (IE11), biến không hoạt động — khi cần, đặt một giá trị cố định trước dòng dùng var() để làm phương án dự phòng.

11 Mẹo đặt tên & tổ chức biến Tổng kết

Gom vào :root. Đặt toàn bộ "design token" — màu, spacing, radius, shadow, cỡ chữ — ở một chỗ. Khi cần đổi tông cả trang, bạn chỉ sửa vài dòng.

Đặt tên có ý nghĩa. Ưu tiên tên theo vai trò (--primary, --text, --bg, --radius, --gap) hơn theo giá trị (--xanh-1), để khi đổi màu vẫn giữ nguyên ý nghĩa.

Nhóm theo loại. Tách thành các nhóm: màu (--c-*), khoảng cách (--space-*), bo góc (--radius-*), bóng (--shadow-*) — dễ tra cứu & mở rộng.

Luôn nghĩ tới dự phòng. Với biến có thể thiếu, dùng var(--x, mặc-định) để giao diện không vỡ.

Đừng đổi :root bừa bãi trong component dùng chung. Đổi biến toàn cục sẽ ảnh hưởng mọi nơi. Muốn riêng cho một khu vực, hãy đặt biến cục bộ ngay trên phần tử cha của khu vực đó.
