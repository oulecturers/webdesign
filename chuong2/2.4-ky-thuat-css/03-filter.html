CSS filter — hiệu ứng đồ hoạ

Áp các hiệu ứng như làm mờ, đổi màu, độ sáng/tương phản, đổ bóng theo hình… trực tiếp lên phần tử hoặc ảnh — không cần phần mềm chỉnh sửa. Đi từ blur, grayscale cơ bản đến kết hợp nhiều hàm, hiệu ứng hover và backdrop-filter (kính mờ).

1 filter là gì & cách dùng Cơ bản

Thuộc tính filter áp các hiệu ứng đồ hoạ (graphical effects) lên một phần tử — thường dùng nhất cho ảnh. Bạn truyền cho nó một hoặc nhiều hàm filter như blur(), brightness(), grayscale()…

Kết quả · ảnh gốc và ảnh đã làm xám
Gốc — filter: none
filter: grayscale(100%)

Cú pháp chung: filter: <hàm>(<giá-trị>);. Giá trị thường là % (0%–100% trở lên), px (cho blur) hoặc deg (cho hue-rotate).

Giá trị mặc định là none — không có hiệu ứng nào. Hiệu ứng chỉ làm thay đổi hiển thị, không sửa file ảnh gốc.

/* Áp 1 hàm filter cho ảnh */
img { filter: grayscale(100%); }

/* Bỏ hiệu ứng */
img { filter: none; }
Đơn vị giá trị thường gặp
Đơn vịDùng choÝ nghĩa
%brightness, contrast, grayscale, sepia, saturate, invert, opacity0% = không có hiệu ứng (với grayscale/sepia/invert) hoặc tối/trắng đen tuỳ hàm; 100% = áp tối đa.
pxblur()Bán kính làm mờ — càng lớn càng mờ.
deghue-rotate()Số độ xoay vòng tròn màu (0–360deg).

filter hoạt động trên hầu hết phần tử (div, ảnh, video, văn bản…), nhưng phổ biến và đẹp nhất là khi áp cho ảnh. Trình duyệt hiện đại hỗ trợ rất tốt.

2 blur() — làm mờ Cơ bản

blur(px) làm mờ phần tử. Tham số là bán kính mờ tính bằng px: số càng lớn thì càng mờ. blur(0) = sắc nét (mặc định).

Kết quả · 0px → 2px → 5px
blur(0)
blur(2px)
blur(5px) 
img { filter: blur(5px); }

Hay dùng để tạo ảnh nền mờ, hiệu ứng "ảnh đang tải", hoặc làm nổi nội dung phía trên. Nếu cần làm mờ phần phía sau một khối (kính mờ) thì xem mục 9 — backdrop-filter.

3 brightness() & contrast() Cơ bản

brightness(%) chỉnh độ sáng: 100% = gốc, 0% = đen hẳn, >100% = sáng hơn. contrast(%) chỉnh độ tương phản: 100% = gốc, 0% = xám phẳng, >100% = tương phản mạnh.

Kết quả · brightness
brightness(50%) — tối hơn
brightness(100%) — gốc
brightness(180%) — sáng hơn
Kết quả · contrast
contrast(50%) — phẳng
contrast(100%) — gốc
contrast(200%) — gắt
.toi   { filter: brightness(50%); }
.sang  { filter: brightness(180%); }
.tuong-phan { filter: contrast(200%); }

4 grayscale() & sepia() Cơ bản

grayscale(%) chuyển ảnh sang đen trắng (100% = xám hoàn toàn). sepia(%) nhuộm ảnh thành tông nâu cổ điển (kiểu ảnh xưa, 100% = sepia đầy đủ).

Kết quả · gốc → đen trắng → nâu cổ
Gốc
grayscale(100%)
sepia(100%)
.den-trang { filter: grayscale(100%); }
.anh-co    { filter: sepia(100%); }

Có thể đặt giá trị giữa chừng, ví dụ grayscale(60%) để "giảm bão hoà" nhẹ thay vì đen trắng hoàn toàn.

5 hue-rotate · invert · saturate · opacity Cơ bản

Bốn hàm đổi màu nâng cao. Mỗi ảnh dưới đây áp một filter khác nhau để dễ so sánh.

Kết quả · lưới ảnh — mỗi ô một hàm filter
Gốc
hue-rotate(90deg)
hue-rotate(220deg)
invert(100%)
saturate(300%)
saturate(0%) — như xám
opacity(40%)
Ý nghĩa nhanh
HàmTác dụngMốc giá trị
hue-rotate(deg)Xoay bánh xe màu — đổi tông màu tổng thể.0deg = gốc; 360deg = về gốc.
invert(%)Đảo màu (âm bản).0% = gốc; 100% = đảo hoàn toàn.
saturate(%)Độ bão hoà (đậm/nhạt của màu).0% = xám; 100% = gốc; >100% = rực hơn.
opacity(%)Độ trong suốt (giống thuộc tính opacity).0% = trong suốt; 100% = đặc.

filter: opacity() và thuộc tính opacity cho kết quả tương tự, nhưng dạng filter có thể kết hợp với các hàm khác và đôi khi được trình duyệt tăng tốc phần cứng tốt hơn.

6 drop-shadow() — đổ bóng theo hình Cơ bản

drop-shadow(x y blur color) tạo bóng bám theo hình dạng thực của ảnh — kể cả vùng trong suốt. Khác hẳn box-shadow vốn chỉ đổ bóng quanh hộp chữ nhật.

Kết quả · drop-shadow (theo hình) vs box-shadow (theo hộp)
filter: drop-shadow()
bóng ôm theo hình
box-shadow
bóng theo khung vuông 
/* x=8px sang phải, y=8px xuống, mờ 6px, màu đen 50% */
img { filter: drop-shadow(8px 8px 6px rgba(0,0,0,.5)); }

Dùng drop-shadow() cho ảnh PNG/SVG có nền trong suốt (logo, icon, hình cắt nền) để bóng đổ đúng theo viền vật thể — đẹp hơn hẳn box-shadow.

7 Kết hợp nhiều filter Nâng cao

Có thể áp nhiều hàm cùng lúc bằng cách viết liền nhau, cách nhau bởi dấu cách (không dùng dấu phẩy). Trình duyệt áp lần lượt theo thứ tự khai báo.

Kết quả · gốc vs đã "tăng chất"
Gốc
contrast + brightness + saturate
/* Nhiều hàm cách nhau bằng DẤU CÁCH, không phải dấu phẩy */
img {
  filter: contrast(140%) brightness(110%) saturate(160%);
}

Đừng dùng dấu phẩy giữa các hàm: filter: blur(2px), grayscale(50%) là sai. Phải viết filter: blur(2px) grayscale(50%). Thứ tự cũng có thể ảnh hưởng kết quả.

8 filter + transition khi hover Nâng cao

Vì filter có thể chuyển động mượt (animatable), kết hợp với transition tạo hiệu ứng tương tác đẹp: ảnh xám chuyển dần sang màu khi rê chuột.

Kết quả · RÊ CHUỘT vào ảnh để thấy hiệu ứng
Xám → màu khi hover
Xám → màu khi hover 
.anh {
  filter: grayscale(100%);
  transition: filter .45s ease;   /* mượt 0.45s */
}
.anh:hover {
  filter: grayscale(0%);       /* trở lại màu */
}

Mẫu này rất hay dùng cho thư viện ảnh, lưới logo đối tác, hoặc danh sách thành viên: mặc định đen trắng cho đồng bộ, khi hover mới hiện màu.

9 backdrop-filter — kính mờ (glassmorphism) Nâng cao

Khác với filter (tác động lên chính phần tử), backdrop-filter làm hiệu ứng lên NỀN nằm phía sau phần tử. Kết hợp nền bán trong suốt + blur() tạo hiệu ứng kính mờ (frosted glass) rất thịnh hành.

Kết quả · khối kính mờ chồng lên ảnh nền có hoạ tiết

Glassmorphism

backdrop-filter: blur(8px) saturate(160%) — nền ảnh phía sau bị mờ qua lớp kính.

Để thấy rõ hiệu ứng cần ba điều kiện: (1) có một ảnh/hoạ tiết phía sau; (2) khối kính có nền bán trong suốt (rgba(255,255,255,.18)); (3) đặt backdrop-filter: blur(...) cho chính khối kính đó.

.kinh-mo {
  background: rgba(255,255,255,.18);   /* bán trong suốt */
  -webkit-backdrop-filter: blur(8px) saturate(160%); /* Safari/Chrome cũ */
          backdrop-filter: blur(8px) saturate(160%);
  border: 1px solid rgba(255,255,255,.35);
}

Cần kèm tiền tố -webkit-backdrop-filter cho Safari. Nếu khối kính không có nền hoặc nền đặc thì sẽ không thấy hiệu ứng. Hiệu ứng cũng khá nặng trên máy yếu.

11 Mẹo & lưu ý Tổng kết

1. filter ảnh hưởng cả phần tử CON. Đặt filter: blur() lên một <div> thì toàn bộ chữ và ảnh bên trong đều bị mờ theo. Muốn chỉ mờ ảnh thì áp filter trực tiếp lên ảnh.

2. backdrop-filter cần nền phía sau. Phải có hoạ tiết/ảnh phía sau và khối kính có nền bán trong suốt, đồng thời thêm tiền tố -webkit-backdrop-filter cho Safari.

3. Kết hợp nhiều hàm bằng dấu cách, không dùng dấu phẩy; thứ tự có thể ảnh hưởng. filter chuyển động được nên rất hợp với transition/animation.

4. Cân nhắc hiệu năng. blur() và nhất là backdrop-filter tốn tài nguyên; tránh lạm dụng trên vùng lớn hoặc khi cuộn trang nhiều, đặc biệt trên thiết bị yếu.
