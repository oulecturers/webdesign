Ba cách chèn CSS & quy tắc Cascade

1 CSS là gì & cú pháp Cơ bản

CSS (Cascading Style Sheets) là ngôn ngữ mô tả cách trình bày của tài liệu HTML: màu sắc, phông chữ, kích thước, bố cục… HTML lo nội dung, còn CSS lo hình thức. Một quy tắc CSS gồm bộ chọn (selector) và một hoặc nhiều cặp thuộc tính: giá trị đặt trong dấu ngoặc nhọn.

Sơ đồ cú pháp một quy tắc CSS
h1bộ chọn (selector) { colorthuộc tính (property) : bluegiá trị (value) ; }

Đọc là: "Mọi thẻ <h1> sẽ có chữ màu xanh." Cặp thuộc tính: giá trị gọi là một khai báo (declaration), kết thúc bằng dấu chấm phẩy.

Bộ chọn chỉ ra phần tử nào được tạo kiểu (vd h1, .card, #menu).

Khối khai báo nằm trong { }, chứa các khai báo phân tách bằng dấu ;. Mỗi khai báo là một cặp property: value;.

Các thành phần của một quy tắc CSS
Thành phầnVí dụVai trò
Selector (bộ chọn)h1Chọn phần tử cần tạo kiểu
Property (thuộc tính)colorTên đặc tính muốn thay đổi
Value (giá trị)blueGiá trị gán cho thuộc tính
Declaration (khai báo)color: blue;Một cặp property + value, kết bằng ;

Chú thích trong CSS

Chú thích đặt giữa /* và */ — có thể trên một hoặc nhiều dòng, dùng để ghi giải thích. CSS không hỗ trợ kiểu chú thích // như JavaScript.

/* Đây là một chú thích một dòng */
p {
  color: red; /* chú thích cuối dòng cũng hợp lệ */
}

/* Chú thích
   nhiều dòng
   cũng được */

CSS bỏ qua khoảng trắng và xuống dòng, nên có thể viết quy tắc trên một dòng hay nhiều dòng tuỳ ý. Nên thụt lề rõ ràng để dễ đọc.

2 Cách 1 — Inline CSS (thuộc tính style) Cơ bản

Viết CSS ngay trên phần tử qua thuộc tính style. Chỉ ảnh hưởng đúng phần tử đó. Không cần selector vì đã gắn trực tiếp; các khai báo phân tách bằng dấu ;.

Kết quả — màu & nền đặt thẳng trên thẻ

Đoạn này có CSS inline: chữ xanh đậm, nền xanh nhạt, có bo góc.

Đoạn này dùng style inline khác hẳn: chữ trắng, nền đỏ.

<p style="color:#16407a; background:#dbeafe; padding:10px 14px;">
  Đoạn này có CSS inline.
</p>
Ưu & nhược của Inline CSS
Ưu điểmNhược điểm
Nhanh, áp dụng tức thì cho 1 phần tửTrộn lẫn nội dung & trình bày → khó đọc, khó bảo trì
Độ ưu tiên cao, hữu ích để thử nhanhKhông tái sử dụng được; lặp lại nếu nhiều phần tử
Không cần selectorKhông dùng được pseudo-class (:hover) hay media query

Hạn chế dùng inline trong dự án thật. Nó khó bảo trì và có độ ưu tiên rất cao nên dễ "vô tình đè" các quy tắc khác (xem mục 6 & 8).

3 Cách 2 — Internal CSS (thẻ <style>) Cơ bản

Đặt toàn bộ CSS trong một thẻ <style> nằm ở <head>. Áp dụng cho cả một trang đó. Phù hợp khi trang có style riêng không chia sẻ với trang khác.

Kết quả — quy tắc lấy từ thẻ <style> của trang này

Tiêu đề định kiểu bằng internal CSS

Đoạn này dùng class .d-internal-p được định nghĩa trong <style> ở đầu trang — nền tím nhạt, viền trái tím.

<head>
  <style>
    h1   { color: #6c5ce7; }
    .ghi-chu { background: #ede9fe;
               border-left: 4px solid #6c5ce7; }
  </style>
</head>

Khi nào dùng internal? Trang đơn lẻ, trang demo/landing nhỏ, email HTML, hoặc khi muốn gom tất cả vào một file duy nhất. Một thay đổi áp dụng cho toàn trang nhưng không dùng lại được cho các trang khác.

Internal CSS tách trình bày khỏi từng thẻ (gọn hơn inline) nhưng vẫn nằm chung file với HTML, nên khi website nhiều trang sẽ phải lặp lại ở mỗi file.

4 Cách 3 — External CSS (file .css riêng) Khuyến nghị

Viết CSS trong một file .css riêng rồi nhúng vào HTML bằng thẻ <link> trong <head>. Đây là cách phổ biến và được khuyến nghị: nhiều trang dùng chung một file, sửa một nơi là cả site đổi theo.

Kết quả — nút này định kiểu trong file CSS dùng chung

Chính trang bài học bạn đang đọc dùng external CSS: file ../../assets/lesson.css tạo toàn bộ khung giao diện.

Nút mẫu (.d-ext-btn)

Bước 1 — tạo file CSS riêng, ví dụ main.css

/* main.css — file CSS ngoài, KHÔNG có thẻ HTML nào */
body { font-family: Arial, sans-serif; }
h1   { color: #1D559F; }
.btn { background: #04AA6D; color: #fff; }

Bước 2 — nhúng vào HTML bằng <link>

<head>
  <link rel="stylesheet" href="main.css">
</head>
Các thuộc tính của thẻ <link> tải CSS
Thuộc tínhGiá trịÝ nghĩa
relstylesheetCho biết file liên kết là bảng định kiểu
hrefmain.cssĐường dẫn tới file CSS (tương đối hoặc tuyệt đối)
type (tuỳ chọn)text/cssKiểu MIME; HTML5 có thể bỏ qua
media (tuỳ chọn)print, screen…Chỉ áp dụng cho thiết bị tương ứng

File .css ngoài không chứa thẻ HTML, không có <style> — chỉ có các quy tắc CSS thuần. Đường dẫn href có thể là tương đối (css/main.css, ../../assets/style.css) hoặc tuyệt đối (URL đầy đủ).

Vì sao nên ưu tiên external? Tách hẳn nội dung (HTML) khỏi trình bày (CSS); nhiều trang dùng chung; trình duyệt cache file CSS nên tải nhanh hơn ở lần sau; sửa một chỗ, cả site cập nhật.

Cả ba cách đều hợp lệ. Chọn cách nào tuỳ phạm vi áp dụng và khả năng bảo trì.

Có thể kết hợp cả ba trong một trang. Khi đó nếu cùng nhắm một phần tử mà xung đột, kết quả do cascade & specificity quyết định — xem các mục tiếp theo.

6 Cascade — thứ tự xếp chồng & ưu tiên Nâng cao

"Cascading" (xếp chồng) nghĩa là khi nhiều quy tắc cùng nhắm một phần tử, trình duyệt áp dụng tất cả rồi chọn người thắng cho mỗi thuộc tính theo các nguyên tắc: (1) độ quan trọng (!important), (2) độ ưu tiên (specificity), (3) thứ tự khai báo (cái viết sau đè cái trước).

Cùng class .d-cas có 2 quy tắc — quy tắc viết sau thắng

Tôi được khai báo nền xanh rồi nền vàng. Vì hai quy tắc cùng độ ưu tiên, quy tắc viết sau (nền vàng) thắng.

.d-cas { background: #dbeafe; } /* quy tắc TRƯỚC — bị đè */
.d-cas { background: #fde68a; } /* quy tắc SAU  — THẮNG  */

Thứ tự ưu tiên giữa 3 cách chèn (khi cùng specificity): inline thắng internal/external, còn internal vs external thì cái nào khai báo sau trong tài liệu sẽ thắng. Thông thường người ta đặt <link> rồi mới tới <style> nên internal đè external.

Thứ tự cascade tổng quát (từ thấp → cao)
Mức ưu tiênNguồn quy tắc
1 (thấp nhất)Mặc định của trình duyệt
2External CSS (<link>)
3Internal CSS (<style>) — nếu đặt sau link
4Inline CSS (thuộc tính style)
5 (cao nhất)!important

Nguyên tắc dễ nhớ: !important > inline > specificity cao > specificity thấp > quy tắc viết sau (khi mọi thứ ngang nhau).

7 Tính kế thừa (inheritance) Nâng cao

Một số thuộc tính (chủ yếu về văn bản: color, font-family, font-size, line-height…) tự truyền xuống các phần tử con. Nhờ đó chỉ cần khai báo một lần ở phần tử cha (vd body).

Con thừa hưởng color & font của cha; .reset dùng initial để gỡ
Đoạn cha: chữ xanh, font Georgia in nghiêng.
Hộp con: tự kế thừa màu xanh & font nghiêng của cha, dù không khai báo gì.

Đoạn này dùng color: initial và font-style: initial → trở về mặc định (chữ đen, không nghiêng).

Kế thừa & hai từ khoá điều khiển
Loại / từ khoáÝ nghĩa
Thuộc tính kế thừacolor, font-*, line-height, text-align, visibility…
Thuộc tính không kế thừamargin, padding, border, background, width…
inheritÉp phần tử lấy giá trị của cha (kể cả thuộc tính vốn không kế thừa)
initialĐưa thuộc tính về giá trị mặc định gốc của CSS
 
.con   { color: inherit; }  /* ép lấy màu của phần tử cha */
.go-bo { color: initial; } /* trả color về mặc định (đen) */

Kế thừa giúp viết CSS ngắn gọn: đặt font-family một lần ở body, toàn trang dùng chung. Muốn cắt kế thừa cho một nhánh, dùng initial hoặc đặt giá trị mới.

8 Độ ưu tiên — Specificity Nâng cao

Khi nhiều quy tắc cùng độ quan trọng nhắm một phần tử, quy tắc có specificity cao hơn thắng. Quy tắc nhớ: id > class > element.

Cùng 1 đoạn bị 3 selector nhắm — id (#d-spec) thắng → chữ đỏ

Đoạn này khớp cả p (tím), .d-spec-cls (xanh lá) và #d-spec (đỏ) → #id có điểm cao nhất nên chữ hiển thị đỏ.

Cách tính điểm (a, b, c)

Mỗi selector quy thành bộ ba điểm (a, b, c), so từ trái sang phải:

Quy đổi điểm specificity
CộtĐếm thành phầnVí dụ
aSố #id#menu → (1,0,0)
bSố .class, [attr], pseudo-class (:hover).btn:hover → (0,2,0)
cSố tên thẻ & pseudo-element (::before)ul li a → (0,0,3)
So điểm các selector nhắm cùng phần tử ở demo trên
SelectorĐiểm (a,b,c)Kết quả
p(0,0,1)thua bị đè
.d-spec-cls(0,1,0)thua bị đè
#d-spec(1,0,0)thắng áp dụng

So sánh theo thứ tự a → b → c: (1,0,0) luôn lớn hơn (0,99,99). Một #id mạnh hơn bất kỳ số lượng class nào. Inline còn được tính như cột cao hơn nữa.

9 !important — đè mọi thứ Nâng cao

Thêm !important sau giá trị làm khai báo đó thắng tất cả — kể cả inline và specificity cao hơn. Là "lá bài tẩy" nhưng nên hạn chế vì phá vỡ trật tự cascade tự nhiên.

Inline đặt màu xanh dương, nhưng !important (xanh lá) vẫn thắng

Đoạn này có inline color:#1D559F (xanh dương) nhưng class dùng color:#04AA6D !important → hiển thị xanh lá.

/* CSS */
.d-imp-strong { color: #04AA6D !important; } /* đè cả inline */

<!-- HTML -->
<p class="d-imp-strong" style="color:#1D559F"> ... </p>
Trật tự ưu tiên đầy đủ (cao nhất → thấp nhất)
HạngQuy tắc
1!important (trong CSS)
2Inline style
3#id
4.class · [attr] · :pseudo-class
5Tên thẻ · ::pseudo-element

Vì sao hạn chế !important? Nó "đóng băng" giá trị, khiến về sau rất khó ghi đè (phải dùng tiếp một !important khác có specificity cao hơn). Code trở nên rối, khó gỡ lỗi. Chỉ nên dùng khi thật cần (vd ghi đè CSS của thư viện bên thứ ba).

10 Khuyến nghị & tổng kết Tổng kết

Ưu tiên External CSS. Đặt CSS trong file .css riêng nhúng qua <link>: tách nội dung khỏi trình bày, tái sử dụng cho nhiều trang, dễ bảo trì và được cache.

Hạn chế Inline. Chỉ dùng để thử nhanh; tránh trộn style="..." khắp nơi vì khó bảo trì và có độ ưu tiên cao gây khó ghi đè.

Tránh lạm dụng !important. Hãy giải quyết xung đột bằng selector rõ ràng và specificity hợp lý thay vì "ép" bằng !important.

Ghi nhớ luồng quyết định khi xung đột: !important → inline → specificity (id > class > element) → quy tắc viết sau. Còn các thuộc tính văn bản thì kế thừa xuống con nếu không khai báo lại.
