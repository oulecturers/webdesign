CSS Selector — bộ chọn phần tử

Selector là "địa chỉ" cho biết quy tắc CSS sẽ áp vào phần tử nào. Nắm vững selector giúp bạn nhắm trúng đối tượng, viết CSS gọn và dễ bảo trì. Đi từ element/class/id cơ bản đến combinator, pseudo-class, pseudo-element và cách tính độ ưu tiên.

1 Selector cơ bản Cơ bản

Năm bộ chọn nền tảng: theo tên thẻ, theo id (#), theo class (.), tất cả phần tử (*) và nhóm nhiều bộ chọn (ngăn bằng dấu phẩy).

Kết quả

Tiêu đề (nhóm A, B tô tím)

Mọi <p> ở đây tô xanh đậm (selector theo tên thẻ).

Đoạn có class .hot tô đỏ đậm.

Đoạn có id #d-special nền vàng.

universal * tô viền mọi con 
/* theo tên thẻ — mọi <p> */
p        { color: #16407a; }
/* theo id — duy nhất trên trang, dùng dấu # */
#special { background: #fde68a; }
/* theo class — tái sử dụng nhiều nơi, dùng dấu . */
.hot     { color: #E63946; }
/* universal — chọn TẤT CẢ phần tử */
*        { box-sizing: border-box; }
/* nhóm — áp cùng quy tắc cho nhiều selector */
h4, figcaption { color: #6c5ce7; }
Năm selector cơ bản
SelectorVí dụChọn gì
Tên thẻ (element)pMọi phần tử <p>.
id#navPhần tử có id="nav" (chỉ một trên trang).
class.btnMọi phần tử có class="btn".
Kết hợp thẻ + classa.btnPhần tử <a> có class btn.
Universal*Tất cả phần tử.
Nhómh1, h2, h3Mọi h1 hoặc h2 hoặc h3.

id nên là duy nhất trong một trang; còn class được phép lặp lại nhiều lần. Vì vậy hãy ưu tiên class để dễ tái sử dụng (xem mục 10).

2 Combinator — kết hợp bộ chọn Cơ bản

Combinator mô tả quan hệ giữa các phần tử: hậu duệ (khoảng trắng), con trực tiếp (>), anh em kề liền (+) và anh em chung sau đó (~).

2.1 — Hậu duệ A B & con trực tiếp A > B

Kết quả

Hậu duệ: em này được tô (nằm sâu bên trong).

Và em lồng sâu hơn cũng được tô.

Con trực tiếp: tôi tô xanh lá (là > p).

Tôi KHÔNG tô vì là cháu, không phải con trực tiếp.

2.2 — Kề liền A + B & anh em chung A ~ B

Kết quả
Kề liền (+):

Tôi NGAY SAU h5 → được tô vàng.

Tôi đứng xa hơn → không tô.

Anh em chung (~):

p sau h5 → tô tím.

p này cũng sau h5 → tô tím luôn.

/* hậu duệ: mọi em BÊN TRONG .menu (con, cháu, chắt…) */
.menu em      { color: green; }
/* con trực tiếp: chỉ p là CON ngay của .menu */
.menu > p      { color: green; }
/* kề liền: p đứng NGAY SAU h5 (cùng cha) */
h5 + p        { background: yellow; }
/* anh em chung: MỌI p sau h5 (cùng cha) */
h5 ~ p        { color: purple; }
Bốn combinator
Cú phápTênÝ nghĩa
A B (khoảng trắng)Hậu duệB nằm bên trong A ở bất kỳ độ sâu nào.
A > BCon trực tiếpB là con ngay của A (một cấp).
A + BAnh em kề liềnB đứng ngay sau A, cùng một cha.
A ~ BAnh em chungMọi B đứng sau A (cùng cha), không cần kề.

3 Selector thuộc tính Nâng cao

Chọn phần tử dựa trên thuộc tính và giá trị của nó. Rất mạnh để tạo kiểu cho liên kết, ô input, hay nút theo dữ liệu (data-*).

Bảy dạng selector thuộc tính (theo W3Schools)
Cú phápKhớp khiVí dụ
[attr]Có thuộc tính attr (bất kể giá trị).a[target]
[attr="val"]Giá trị đúng bằng val.input[type="text"]
[attr^="val"]Giá trị bắt đầu bằng val.a[href^="https"]
[attr$="val"]Giá trị kết thúc bằng val.a[href$=".pdf"]
[attr*="val"]Giá trị chứa chuỗi val.a[href*="khoa"]
[attr~="val"]Giá trị là danh sách cách nhau bởi dấu cách và chứa từ val.[data-tag~="hot"]
[attr|="val"]Giá trị bằng val hoặc bắt đầu bằng val- (hay dùng cho mã ngôn ngữ).[hreflang|="vi"]
Kết quả · mỗi liên kết khớp một dạng selector
[target] + ^=https (viền xanh, nền nhạt) $=.pdf (nền hồng) *=khoa (in đậm) |=vi (viền vàng) ~=hot (chữ đỏ) 
/* có thuộc tính target */
a[target]            { border-color: green; }
/* href bắt đầu bằng https → liên kết an toàn */
a[href^="https"]   { background: #eafaf3; }
/* href kết thúc .pdf → tài liệu tải về */
a[href$=".pdf"]    { background: #fdecee; }
/* href chứa chuỗi "khoa" */
a[href*="khoa"]    { font-weight: 700; }
/* hreflang = vi hoặc vi-VN, vi-... */
a[hreflang|="vi"]  { outline: 2px solid gold; }
/* data-tag là 1 list, chứa đúng từ "hot" */
[data-tag~="hot"]    { color: red; }

Thêm chữ i trước dấu ngoặc đóng để bỏ qua hoa/thường: a[href$=".PDF" i] khớp cả .pdf lẫn .PDF.

4 Pseudo-class trạng thái Cơ bản

Pseudo-class (lớp giả) bắt đầu bằng một dấu hai chấm :, mô tả trạng thái của phần tử: đang rê chuột, đang focus, đang bấm, đã/chưa thăm.

Kết quả · rê chuột / bấm / focus để thấy đổi màu

Liên kết: rê chuột vào tôi hoặc liên kết ngoài — đổi màu theo trạng thái.

a:link    { color: #1D559F; }  /* chưa thăm */
a:visited { color: #6c5ce7; }  /* đã thăm   */
a:hover   { color: #E63946; }  /* đang rê chuột */
a:active  { color: #FFC107; }  /* đang bấm  */
input:focus { border-color: #04AA6D; } /* đang được chọn */
Pseudo-class trạng thái thường gặp
Pseudo-classKích hoạt khi
:linkLiên kết chưa được truy cập.
:visitedLiên kết đã được truy cập.
:hoverCon trỏ chuột đang ở trên phần tử.
:focusPhần tử đang được chọn (ô nhập, nút, link qua phím Tab).
:activePhần tử đang được bấm giữ.

Thứ tự khai báo cho liên kết phải đúng LoVe – HAte: :link → :visited → :hover → :active. Sai thứ tự thì hiệu ứng hover/active có thể không hiện.

5 Pseudo-class cấu trúc Nâng cao

Chọn phần tử theo vị trí của nó trong cây HTML mà không cần thêm class: mục đầu/cuối, mục thứ n, mục chẵn/lẻ…

Danh sách · :first/:last/:nth-child
  • Mục 1 — :first-child (xanh)
  • Mục 2 — :nth-child(even)
  • Mục 3 — :nth-child(3n) đỏ
  • Mục 4 — :nth-child(even)
  • Mục 5
  • Mục 6 — even + 3n
  • Mục 7 — :last-child (vàng)
Theo loại thẻ · :first-of-type / :nth-of-type

p đầu tiên → xanh lá (:first-of-type)

p thứ hai → bình thường

span 1span 2span 3span 4

span lẻ (1,3) tô tím qua :nth-of-type(odd)

li:first-child     { background: #dbeafe; } /* mục đầu */
li:last-child      { background: #fde68a; } /* mục cuối */
li:nth-child(even) { background: #f6f9fe; } /* dòng chẵn — kẻ vằn */
li:nth-child(3n)   { color: red; }       /* mỗi 3 mục: 3,6,9… */
p:first-of-type   { color: green; }     /* <p> đầu tiên trong cha */
span:nth-of-type(odd){ background: #c4b5fd; } /* span lẻ */
Cú pháp tham số của :nth-child(n)
Tham sốChọn các mục
:nth-child(3)Đúng mục thứ 3.
:nth-child(odd) / (2n+1)Lẻ: 1, 3, 5, 7…
:nth-child(even) / (2n)Chẵn: 2, 4, 6, 8…
:nth-child(3n)Bội của 3: 3, 6, 9…
:nth-child(3n+1)1, 4, 7, 10… (cứ cách 3).
:only-childPhần tử là con duy nhất của cha.

Khác biệt mấu chốt: :nth-child đếm mọi anh em rồi xét vị trí, còn :nth-of-type chỉ đếm các anh em cùng loại thẻ. Hộp dưới minh hoạ :only-child:

:only-child · ul chỉ có 1 li → tô xanh lá
  • Tôi là con duy nhất

6 Pseudo-class khác: phủ định, form & gốc Nâng cao

Nhóm pseudo-class hữu ích cho biểu mẫu và loại trừ: :not(), :checked, :disabled, :required, :empty, :root.

:not() — mọi mục TRỪ .skip
  • Mục thường (tô xanh)
  • Mục .skip (KHÔNG tô)
  • Mục thường (tô xanh)
:empty — phần tử rỗng tô khác
Có nội dung.
Form · :required (viền xanh) · :disabled (xám) · :checked
li:not(.skip)   { background: #eafaf3; }  /* loại trừ .skip */
input:required { border-color: blue; }    /* ô bắt buộc */
input:disabled { background: #e2e8f0; }  /* ô bị khoá  */
input:checked + span { color: green; }  /* checkbox đã tick */
div:empty      { background: #fdecee; }  /* không có con/nội dung */
:root          { --blue: #1D559F; }     /* phần tử gốc <html> — nơi khai báo biến CSS */

:root trỏ tới phần tử gốc <html> nhưng có độ ưu tiên cao hơn, là nơi lý tưởng để khai báo biến CSS (custom properties) dùng chung cho cả trang.

7 Pseudo-element Nâng cao

Pseudo-element (phần tử giả) dùng hai dấu hai chấm ::, tạo kiểu cho một phần của phần tử hoặc chèn nội dung mà không cần thêm thẻ HTML.

::before / ::after chèn nội dung

Chèn icon trước và dấu tick sau

Mỗi dòng tự có 👉 và ✓

::first-letter / ::first-line

Chữ cái đầu được phóng to kiểu báo in (drop cap); còn cả dòng đầu tiên được tô tím và viết kiểu small-caps. Phần còn lại của đoạn văn giữ nguyên định dạng bình thường.

::selection · ::placeholder · ::marker

Bôi đen thử đoạn này — vùng chọn sẽ có nền vàng (::selection).

  • Dấu đầu dòng đổi thành ➤ đỏ (::marker)
  • Mục thứ hai cũng vậy
p::before  { content: "👉 "; }          /* chèn trước nội dung */
p::after   { content: " ✓"; color: green; }
p::first-letter { font-size: 2.4em; float: left; }
p::first-line   { font-variant: small-caps; }
::selection      { background: gold; }       /* vùng bôi đen */
input::placeholder { color: #94a3b8; }    /* chữ gợi ý */
li::marker     { color: red; content: "➤ "; } /* chấm/số đầu dòng */

::before và ::after bắt buộc phải có thuộc tính content (dù chỉ là content: "") thì mới hiển thị. Nội dung chèn là "ảo", không vào được DOM.

8 Độ ưu tiên — Specificity Nâng cao

Khi nhiều quy tắc cùng nhắm một phần tử, trình duyệt chọn quy tắc có độ ưu tiên cao nhất. Tính điểm theo bộ ba (id, class, element) — so từ trái sang phải.

Cách tính điểm specificity
Loại selectorCộng vàoVí dụ → điểm
Inline style (style="…")Cao nhất (1,0,0,0)luôn thắng selector thường
id (#nav)+1 cột id#box → (1,0,0)
class / pseudo-class / thuộc tính+1 cột class.menu:hover → (0,2,0)
element / pseudo-element+1 cột elementp::before → (0,0,2)
universal *+0 (không cộng gì)* → (0,0,0)
Ai thắng? · cùng nhắm vào một đoạn <p id="x" class="y">

Đoạn này thực tế đang nhận màu nền của quy tắc thắng cuộc.

Giả sử có 3 quy tắc cùng đặt background cho đoạn trên:

p          { background: #eee; }    /* (0,0,1) = 1 điểm element  */
.d-win     { background: #fde68a; } /* (0,1,0) = 1 điểm class    → mạnh hơn element */
#d-win     { background: #bbf7d0; } /* (1,0,0) = 1 điểm id       → MẠNH NHẤT, thắng */
So điểm vài selector (đọc như số: id > class > element)
Selector(id, class, element)Mạnh hơn?
p(0, 0, 1)yếu nhất
.btn(0, 1, 0)> p
nav a.btn(0, 1, 2)> .btn
#menu .btn(1, 1, 0)> mọi cái không có id
style="…"(1,0,0,0)thắng gần như tất cả

!important ghi đè cả specificity — nhưng hạn chế dùng vì nó phá vỡ quy tắc tự nhiên, khiến CSS rất khó bảo trì và gỡ lỗi. Hãy ưu tiên viết selector hợp lý trước.

10 Mẹo viết selector tốt Tổng kết

Mẹo 1 — Ưu tiên class hơn id. Class tái sử dụng được nhiều nơi và có specificity thấp hơn nên dễ ghi đè khi cần. Hãy để dành id cho mỏ neo (#section) hay JavaScript.

Mẹo 2 — Đừng lồng selector quá sâu. Kiểu .a .b .c .d .e span vừa khó đọc, vừa khó ghi đè, vừa chậm. Giữ selector nông và phẳng, đặt một class mô tả ngay trên phần tử đích.

Mẹo 3 — Một dấu : cho pseudo-class, hai dấu :: cho pseudo-element. :hover (lớp giả) khác ::before (phần tử giả). Trình duyệt vẫn chấp nhận :before kiểu cũ, nhưng nên dùng ::before cho đúng chuẩn.

Mẹo 4 — Tránh lạm dụng !important và universal *. !important phá trật tự ưu tiên; * { … } áp lên mọi phần tử có thể gây chậm và lỗi ngoài ý muốn.

Ghi nhớ thứ tự ưu tiên: inline style > #id > .class / [attr] / :pseudo-class > element / ::pseudo-element > *.
