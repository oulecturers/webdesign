Bố cục hiện đại: Flexbox & Grid

Hai cỗ máy dàn trang mạnh nhất của CSS hiện đại. Flexbox dàn theo một chiều (hàng hoặc cột), Grid dàn theo hai chiều (hàng × cột). Kết bài bằng hai sản phẩm thực tế: thanh điều hướng (navbar) và biểu mẫu (form) đẹp hoàn toàn bằng CSS.

1 display:flex — container, item & hai trục Cơ bản

Khi đặt display:flex cho một phần tử, nó trở thành flex container; mọi con trực tiếp của nó tự động thành flex item và xếp thành một hàng. Mọi thứ trong Flexbox xoay quanh hai trục: trục chính (main axis) và trục phụ (cross axis).

Kết quả · 3 item xếp theo trục chính (mặc định là hàng ngang)
1 2 3

Trục chính (main axis) là hướng các item xếp ra. Mặc định flex-direction:row nên trục chính nằm ngang (trái → phải). justify-content căn item theo trục này.

Trục phụ (cross axis) vuông góc với trục chính (mặc định là dọc). align-items căn item theo trục này.

/* Cha thành flex container — con tự thành flex item */
.container {
  display: flex;
  gap: 10px;   /* khoảng cách giữa các item */
}
Hai nhóm thuộc tính trong Flexbox
Đặt ở đâuThuộc tính tiêu biểuTác dụng
Trên containerflex-direction, flex-wrap, justify-content, align-items, align-content, gapĐiều khiển cách cả nhóm con dàn ra.
Trên item (con)order, flex-grow, flex-shrink, flex-basis, flex, align-selfĐiều khiển riêng từng con.

Mẹo căn giữa thần thánh. Muốn căn giữa một phần tử cả ngang lẫn dọc: cho cha display:flex; justify-content:center; align-items:center; — xong!

4 align-items, align-content & gap Cơ bản

align-items căn các item theo trục phụ (mặc định là chiều dọc — bên trong chiều cao container). align-content căn nhiều dòng khi đã wrap. gap tạo khoảng cách giữa các item.

4.2 — gap: khoảng cách giữa các item

gap: 16px 24px → 16px giữa các hàng, 24px giữa các cột
123 456

4.3 — align-content: căn các DÒNG khi đã wrap Nâng cao

Container cao 170px, wrap thành 2 dòng · align-content: space-between
123 45 
.container {
  display: flex;
  align-items: center;       /* căn item theo trục phụ (1 dòng) */
  align-content: space-between; /* căn nhiều DÒNG (cần flex-wrap:wrap) */
  gap: 16px 24px;          /* hàng 16px · cột 24px */
}

align-items căn các item trong một dòng; align-content chỉ có tác dụng khi có nhiều dòng (đã flex-wrap:wrap) và container còn dư chiều cao.

5 Thuộc tính của flex item Nâng cao

Đặt trên từng con để điều khiển thứ tự, độ co giãn và canh lề riêng: order, flex-grow, flex-shrink, flex-basis, viết tắt flex, và align-self.

5.1 — order: đổi thứ tự hiển thị (không đổi HTML)

HTML viết 1·2·3·4 nhưng order đẩy "4" lên đầu, "1" xuống cuối
1 2 3 4

5.2 — flex-grow: chia phần dư theo tỉ lệ

grow 0 · 1 · 2 → ô thứ ba "ngốn" gấp đôi phần dư so với ô giữa
grow:0 grow:1 grow:2

5.3 — flex-basis & viết tắt flex

Ô trái cố định 200px (flex: 0 0 200px), ô phải lấp phần còn lại (flex: 1)

5.4 — align-self: ghi đè align-items cho riêng 1 item

Cả nhóm canh trên (flex-start), riêng ô giữa align-self: flex-end
trên tôi xuống đáy trên 
/* flex là viết tắt của: flex-grow flex-shrink flex-basis */
.item {
  flex: 1;            /* = 1 1 0  → các item chia đều phần dư */
  flex: 0 0 200px;     /* không co/giãn, rộng cố định 200px */
  order: -1;          /* số nhỏ ra trước (mặc định 0) */
  align-self: flex-end; /* canh riêng item này theo trục phụ */
}

6 display:grid — cột/hàng & đơn vị fr Cơ bản

Grid dàn theo hai chiều: bạn khai báo số cột bằng grid-template-columns và (tuỳ chọn) số hàng bằng grid-template-rows. Đơn vị fr (fraction) chia phần dư theo tỉ lệ.

6.1 — Ba cột bằng nhau với 1fr 1fr 1fr

grid-template-columns: 1fr 1fr 1fr
123 456

6.2 — Trộn px và fr

grid-template-columns: 120px 1fr 80px → cột giữa co giãn, hai cột biên cố định
120px1fr (co giãn)80px

6.3 — Khai báo cả hàng

columns: 1fr 1fr · rows: 60px 100px
cao 60pxcao 60px cao 100pxcao 100px 
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* 3 cột bằng nhau */
  grid-template-rows: 60px 100px;     /* 2 hàng cao khác nhau */
  gap: 10px;
}

fr = "fraction" (phần). 2fr 1fr nghĩa là cột đầu rộng gấp đôi cột sau. Khác với %, fr tự trừ gap nên không bao giờ tràn.

7 repeat() · minmax() · auto-fit/auto-fill Nâng cao

Bộ ba làm nên lưới responsive chỉ với một dòng CSS — không cần media query. repeat() lặp khai báo, minmax() đặt min–max cho cột, auto-fit tự tính số cột vừa khít.

7.2 — auto-fit: lưới thẻ tự xuống dòng (THU NHỎ cửa sổ để thấy rõ)

  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(140px, 1fr));

Cú phápÝ nghĩa
repeat(3, 1fr)Lặp 1fr ba lần = 1fr 1fr 1fr.
minmax(150px, 1fr)Cột rộng ít nhất 150px, nhiều nhất là 1 phần dư.
auto-fitTự tính số cột vừa khít; cột trống bị co lại (thẻ giãn lấp đầy).
auto-fillNhư auto-fit nhưng giữ cột trống (thẻ không giãn ra).

Khác biệt cốt lõi: auto-fit "thu" cột trống nên các thẻ nở ra chiếm hết hàng; auto-fill "giữ" cột trống nên có thể để lại khoảng trống bên phải.

8 gap & căn ô: justify-items / align-items Cơ bản

Trong Grid, gap (gồm row-gap và column-gap) tạo rãnh giữa các ô. justify-items căn nội dung trong ô theo hàng ngang, align-items theo chiều dọc.

8.1 — row-gap & column-gap riêng biệt

row-gap: 20px · column-gap: 8px
123 456

8.2 — justify-items / align-items: căn nội dung trong từng ô

Ô rộng nhưng item chỉ 80px · justify-items:center · align-items:center
123 
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  row-gap: 20px;
  column-gap: 8px;     /* = gap: 20px 8px */
  justify-items: center; /* căn ngang nội dung trong mỗi ô */
  align-items: center;   /* căn dọc nội dung trong mỗi ô */
}

Phân biệt: justify-items/align-items căn nội dung trong từng ô; còn justify-content/align-content căn cả lưới khi lưới nhỏ hơn container.

9 grid-template-areas — bố cục cả trang Nâng cao

Cách trực quan nhất để dàn cả trang: đặt tên cho từng vùng rồi "vẽ" sơ đồ bố cục bằng chữ. Mỗi con chỉ cần khai báo nó thuộc vùng nào qua grid-area.

Layout kinh điển: header / sidebar / main / footer
HEADER
SIDEBAR
MAIN — nội dung chính
FOOTER
.layout {
  display: grid;
  grid-template-columns: 200px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
}
.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

Tên lặp lại trên nhiều ô = vùng đó trải rộng (như header chiếm cả 2 cột). Dùng dấu chấm . để chừa một ô trống. Khi đổi bố cục cho mobile, chỉ cần viết lại grid-template-areas trong media query — cực gọn.

10 Grid item: grid-column / grid-row / span Nâng cao

Mỗi ô có thể chiếm nhiều cột hoặc nhiều hàng bằng grid-column/grid-row, hoặc dùng từ khoá span N để "trải ra N ô". Đây là sức mạnh tạo lưới không đều.

.item-wide { grid-column: span 2; }      /* chiếm 2 cột */
.item-tall { grid-row: span 2; }         /* chiếm 2 hàng */
.item-big  { grid-column: 1 / span 2;      /* từ đường 1, trải 2 cột */
             grid-row: span 2; }         /* và 2 hàng → ô 2×2 */
Cú phápÝ nghĩa
grid-column: span 2Trải qua 2 cột tính từ vị trí hiện tại.
grid-column: 1 / 3Từ đường dọc số 1 đến số 3 (= 2 cột).
grid-row: span 2Trải qua 2 hàng.
grid-area: 1 / 1 / 3 / 3Viết tắt: row-start / col-start / row-end / col-end.

"Đường" (grid line) được đánh số bắt đầu từ 1. Lưới 3 cột có 4 đường dọc. grid-column: 1 / -1 nghĩa là "từ đường đầu đến đường cuối" — trải hết chiều ngang.

11 Thực hành: Navbar ngang đầy đủ bằng flex Thực hành

Ghép tất cả kiến thức Flexbox thành một thanh điều hướng thật: logo bên trái, menu được đẩy sang phải bằng margin-left:auto, mục đang chọn có lớp .active, các mục có hiệu ứng :hover.

12 Thực hành: Form đẹp định dạng bằng CSS Thực hành

Mặc định form khá xấu. Với vài quy tắc CSS — input rộng 100%, bo góc, hiệu ứng :focus, nút bấm nổi bật — ta có biểu mẫu chuyên nghiệp. Dùng flex để xếp hai ô trên cùng một hàng.

Kết quả · bấm vào ô nhập để thấy viền sáng khi :focus

Chúng tôi sẽ không chia sẻ email của bạn.

.form input,
.form select,
.form textarea {
  width: 100%;            /* ô trải hết chiều ngang */
  padding: 11px 13px;
  border: 1px solid #c7d2e2;
  border-radius: 9px;
}
.form input:focus {                  /* viền sáng khi đang gõ */
  border-color: #1D559F;
  box-shadow: 0 0 0 3px rgba(29,85,159,.18);
  outline: none;
}
.form .row2 { display: flex; gap: 12px; } /* 2 ô cùng hàng */
.form .row2 > .field { flex: 1; }

Luôn gắn <label for="id"> khớp với id của ô nhập để bấm vào nhãn cũng focus được ô — tốt cho khả năng truy cập (accessibility). Đừng quên outline:none phải đi kèm một dấu hiệu focus khác (như box-shadow).

14 Quy tắc chọn: Flex hay Grid? Tổng kết

Một câu để nhớ cả đời, đủ cho 90% tình huống thực tế.

Quy tắc vàng. Cần dàn một dãy phần tử theo một chiều (một hàng nút, một thanh menu, một danh sách card cùng hàng) → dùng display:flex. Cần dàn một khung theo hai chiều (bố cục cả trang với header/sidebar/main/footer, bảng điều khiển, thư viện ảnh nhiều hàng nhiều cột) → dùng display:grid.

Kết hợp cả hai là chuẩn mực. Thực tế người ta thường dùng Grid để dựng khung tổng thể của trang, rồi dùng Flex bên trong mỗi vùng (ví dụ navbar trong header) để căn chỉnh từng dãy phần tử.

Đừng lạm dụng. Không cần Grid để xếp 3 cái nút cạnh nhau — Flex gọn hơn. Ngược lại đừng "vật lộn" với Flex để dựng layout 2 chiều — Grid sinh ra cho việc đó.

Lỗi hay gặp. Quên rằng justify-content/align-items đặt trên container, còn order/flex/grid-column đặt trên item. Đặt nhầm chỗ là thuộc tính sẽ "không có tác dụng".
