1 Kiểu dấu đầu dòng: list-style-type Cơ bản

Thuộc tính list-style-type đổi ký hiệu đầu dòng của danh sách: chấm tròn đặc/rỗng, ô vuông, số, số La Mã, chữ cái… hoặc bỏ hẳn bằng none. Dùng được cho cả <ul> lẫn <ol>.

2 Vị trí dấu, dấu bằng ảnh & viết gộp Cơ bản

list-style-position quyết định dấu nằm ngoài hay trong hộp nội dung; list-style-image thay dấu bằng một ảnh; còn list-style là cách viết gộp cả ba.

  • inside — dấu nằm TRONG khung chữ; khi xuống hàng, chữ chạy thẳng dưới dấu.

2.3 — Viết gộp: list-style

Kết quả · list-style: square inside
  • Gộp type + position trong một dòng
  • Ngắn gọn, dễ đọc
Cú pháp viết gộp list-style
Thành phầnÝ nghĩaVí dụ

/* viết riêng từng thuộc tính */
ul { list-style-type: square;
     list-style-position: inside;
     list-style-image: url("dot.png"); }

/* hoặc gộp 1 dòng */
ul { list-style: square inside url("dot.png"); }

Nếu trình duyệt không tải được ảnh trong list-style-image, nó tự quay về dùng list-style-type. Vì vậy nên đặt cả hai để có "dấu dự phòng".

3 Biến danh sách thành menu ngang Thực hành

Công thức kinh điển của W3Schools: bỏ list-style, xoá margin/padding, rồi cho các <li> nằm cùng hàng bằng display:flex (hiện đại) hoặc display:inline-block (cổ điển).

3.1 — Cách hiện đại: ul { display: flex }

Kết quả · menu flex
  • Trang chủ
  • Giới thiệu
  • Dịch vụ
  • Liên hệ
.menu {
  list-style: none;       /* bỏ chấm đầu dòng   */
  margin: 0; padding: 0;  /* xoá thụt lề mặc định */
  display: flex;          /* các li nằm cùng hàng */
}
.menu li a { display: block; padding: 12px 18px; text-decoration: none; }

3.2 — Cách cổ điển: li { display: inline-block }

Kết quả · menu inline-block

Vì sao đặt display:block cho thẻ <a>? Để cả ô (kể cả phần padding) đều bấm được, chứ không chỉ riêng dòng chữ. Đây là chi tiết nhỏ nhưng tạo trải nghiệm bấm tốt hơn hẳn.

5 Căn lề, chú thích & chiều cao ô Cơ bản

text-align canh ngang, vertical-align canh dọc, height tăng chiều cao hàng, và caption-side đặt tiêu đề bảng (<caption>) ở trên hay dưới.

text-align: left | center | right — canh chữ theo chiều ngang trong ô. Số tiền/điểm thường canh phải, chữ canh trái.

vertical-align: top | middle | bottom — canh nội dung theo chiều dọc khi ô cao. height trên <th>/<td>/<tr> đặt chiều cao tối thiểu.

caption { caption-side: bottom; }   /* hoặc top (mặc định) */
th  { height: 46px; }
td  { text-align: right; vertical-align: middle; }

6 Tô xen kẽ & làm nổi hàng đang trỏ (zebra) Nâng cao

Tô màu xen kẽ các hàng giúp mắt dò ngang dễ hơn rất nhiều. Dùng bộ chọn tr:nth-child(even) / tr:nth-child(odd), kết hợp tr:hover để làm nổi hàng đang trỏ chuột.

Kết quả · zebra + rê chuột vào từng hàng
MãSản phẩmLoạiGiá
SP01Bàn phím cơThiết bị790.000 ₫
SP02Chuột không dâyThiết bị320.000 ₫
SP03Tai ngheÂm thanh1.100.000 ₫
SP04Webcam HDHình ảnh650.000 ₫
SP05Đèn bàn LEDPhụ kiện240.000 ₫
 
/* tô màu các hàng CHẴN (2, 4, 6...) */
tbody tr:nth-child(even) { background: #eafaf3; }

/* làm nổi hàng đang rê chuột */
tbody tr:hover { background: #ffe6a8;
                    transition: background .15s; }

:nth-child(even) = hàng chẵn, :nth-child(odd) = hàng lẻ. Có thể dùng công thức tổng quát như :nth-child(3n) (cứ 3 hàng tô 1) cho hiệu ứng khác.

7 Bảng responsive — bọc overflow-x: auto Nâng cao

Bảng nhiều cột dễ bị tràn khỏi màn hình điện thoại. Giải pháp đơn giản & kinh điển: bọc bảng trong một <div> đặt overflow-x: auto để có thể cuộn ngang bảng mà không phá vỡ bố cục trang.

Kết quả · thu hẹp cửa sổ rồi cuộn ngang bảng
Mã SVHọ và tênNgày sinhLớpEmailĐiện thoạiGPA
2024001Lê Ngọc Trân02/03/2006TH2024Atran.le@ou.edu.vn0901 234 5673.6
2024002Nguyễn Đình Huy15/07/2006TH2024Ahuy.nguyen@ou.edu.vn0902 345 6783.4
2024003Dương Hữu Thành28/11/2005TH2024Bthanh.duong@ou.edu.vn0903 456 7893.8
 
/* HTML: bọc bảng trong 1 div */
<div class="table-wrap">
  <table> ... </table>
</div>

/* CSS */
.table-wrap { overflow-x: auto; }   /* hiện thanh cuộn ngang khi tràn */
table { min-width: 720px; }    /* đảm bảo bảng đủ rộng để cuộn */

Đặt min-width cho bảng để các cột không bị "bóp" quá hẹp. Khi màn hình nhỏ hơn min-width, thanh cuộn ngang xuất hiện thay vì làm vỡ trang.

8 Liên kết: 4 trạng thái theo thứ tự LVHA Cơ bản

Một liên kết <a> có 4 trạng thái, mỗi trạng thái tô màu được riêng. Quan trọng: phải khai báo đúng thứ tự Link → Visited → Hover → Active (ghi nhớ: LoVe HAte), nếu sai thì hover/active bị "nuốt".

Kết quả · rê chuột & giữ chuột để thử đổi màu

Liên kết A Liên kết B Liên kết đã ghé (B) Chỉ gạch chân khi hover

Bốn trạng thái của liên kết (thứ tự bắt buộc)
#Pseudo-classKhi nào
1 · La:linkLiên kết chưa từng được bấm.
2 · Va:visitedLiên kết đã được bấm/đã ghé thăm.
3 · Ha:hoverĐang rê chuột lên trên liên kết.
4 · Aa:activeĐang nhấn giữ (khoảnh khắc bấm xuống).
 
/* PHẢI theo đúng thứ tự L → V → H → A */
a:link    { color: #1D559F; }   /* chưa bấm   */
a:visited { color: #8e44ad; }   /* đã bấm     */
a:hover   { color: #fff; background: #FFC107; } /* rê chuột */
a:active  { color: #fff; background: #E63946; } /* đang giữ */

Sai thứ tự = lỗi thầm lặng. Nếu đặt :hover trước :visited, màu visited sẽ ghi đè và bạn không bao giờ thấy hiệu ứng hover. Luôn nhớ thứ tự LVHA.

9 text-decoration & liên kết kiểu nút Nâng cao

text-decoration: none bỏ gạch chân; thêm display:inline-block, nền, bo góc và transition là biến một liên kết thành nút bấm mượt mà — không cần thẻ <button>.

Kết quả · rê chuột để thấy hiệu ứng nâng & đổi nền
Nút chính Nút xanh lá Nút viền 
.btn {
  display: inline-block;       /* nhận padding/width như block */
  text-decoration: none;       /* bỏ gạch chân */
  color: #fff; background: #1D559F;
  padding: 10px 22px; border-radius: 8px;
  transition: background .2s, transform .2s, box-shadow .2s;
}
.btn:hover {
  background: #16407a;
  transform: translateY(-2px);  /* nhấc lên 2px */
  box-shadow: 0 8px 18px rgba(29,85,159,.3);
}

transition làm thay đổi diễn ra dần trong khoảng thời gian định trước thay vì "nhảy" tức thì — đó là chìa khoá tạo cảm giác mượt khi hover.

text-decoration còn nhận underline, overline, line-through; bản mới cho phép đổi màu/kiểu gạch: text-decoration: underline wavy #E63946.

10 Thanh điều hướng (Navbar) từ <ul> Thực hành

Bài thực hành kinh điển của W3Schools: navbar về bản chất chỉ là một <ul> đã bỏ dấu đầu dòng. Đặt theo chiều ngang (top nav) hoặc dọc (sidebar), và làm nổi mục hiện tại bằng class .active.

10.2 — Navbar DỌC (sidebar)

/* === NAVBAR NGANG === */
ul.topnav { list-style: none; margin: 0; padding: 0;
            overflow: hidden; background: #16407a; }
ul.topnav li   { float: left; }
ul.topnav li a { display: block; color: #fff;
                 padding: 14px 20px; text-decoration: none; }
ul.topnav li a:hover  { background: #1D559F; }
ul.topnav li a.active { background: #04AA6D; }  /* mục hiện tại */

/* === NAVBAR DỌC === */
ul.sidenav   { list-style: none; margin: 0; padding: 0; width: 200px; }
ul.sidenav li a { display: block; padding: 12px 16px; text-decoration: none; }

Class .active đánh dấu trang/mục người dùng đang xem — thường gắn sẵn trong HTML của từng trang. Với navbar ngang dùng float, nhớ thêm clearfix (overflow:hidden hoặc ::after{clear:both}) để khung <ul> không bị "sập".

12 Mẹo & lỗi thường gặp Tổng kết

Mẹo 1 — Menu nên dùng <ul>. Về mặt ngữ nghĩa, một danh sách liên kết chính là một danh sách. Dùng <ul><li><a> rồi bỏ list-style giúp trình đọc màn hình và SEO hiểu đúng cấu trúc điều hướng.

Lỗi 2 — Sai thứ tự LVHA. Luôn khai báo :link → :visited → :hover → :active. Đặt :hover sau :active hoặc trước :visited khiến hiệu ứng bị một quy tắc sau ghi đè và "biến mất".

Mẹo 3 — Zebra giúp đọc bảng dễ hơn. Tô xen kẽ bằng tr:nth-child(even) và làm nổi hàng bằng tr:hover giảm rõ rệt việc "dò nhầm dòng" khi bảng nhiều cột.

Lỗi 4 — Quên border-collapse. Bảng kẻ viền mà thiếu border-collapse: collapse sẽ có viền đôi dày và khe hở. Đây là dòng gần như luôn cần khi tạo bảng.

Ghi nhớ nhanh. Bỏ chấm → list-style:none; menu cùng hàng → flex/inline-block; bảng kẻ đẹp → border-collapse:collapse; link đổi màu → thứ tự LVHA; bảng tràn trên mobile → bọc div{overflow-x:auto}.
