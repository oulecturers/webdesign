Hiệu ứng: transform · transition · animation

Ba nhóm thuộc tính làm web "sống động": biến đổi hình học (di chuyển, xoay, phóng), chuyển trạng thái mượt mà khi tương tác, và hoạt ảnh tự chạy bằng @keyframes. Đi từ cơ bản đến 3D, cubic-bezier và tối ưu hiệu năng.

1 transform: translate — di chuyển Cơ bản

transform biến đổi hình học một phần tử mà không ảnh hưởng tới luồng layout xung quanh. translate dời phần tử theo trục X (ngang) và/hoặc Y (dọc). Viền nét đứt là vị trí gốc.

Kết quả
gốc
chưa biến đổi
translateX
sang phải 28px
translateY
lên trên 18px
translate
phải 24px · xuống 14px

translateX(28px) dời sang phải; số âm dời sang trái. translateY(-18px) dời lên trên (trục Y dương hướng xuống). translate(x, y) gộp cả hai. Giá trị có thể là px, % (theo kích thước chính phần tử), rem…

Các hàm translate
HàmÝ nghĩaVí dụ
translateX(x)Dời theo trục ngangtranslateX(50px)
translateY(y)Dời theo trục dọctranslateY(-20px)
translate(x, y)Dời cả hai trụctranslate(30px, 10px)
translate(x)Chỉ ghi 1 giá trị = chỉ trục Xtranslate(50%)
 
/* dời sang phải 28px */
.box { transform: translateX(28px); }

/* mẹo kinh điển: căn giữa tuyệt đối một phần tử */
.center {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%); /* lùi nửa kích thước chính nó */
}

Vì % trong translate tính theo kích thước của chính phần tử, nên translate(-50%, -50%) là cách căn giữa tuyệt đối phổ biến nhất.

2 rotate · scale · skew & transform-origin Cơ bản

Ngoài dời chỗ, transform còn xoay (rotate), phóng/thu (scale) và nghiêng (skew). transform-origin quyết định "tâm" của phép biến đổi.

scaleX / scaleYPhóng theo từng trụcscaleX(.5)
skewX / skewYNghiêng (xô) theo trụcskewX(20deg)
transform-originĐặt tâm biến đổi (mặc định center)top left · 0 100%
 
.rot   { transform: rotate(30deg); }
.scale { transform: scale(1.35); }
.skew  { transform: skewX(-18deg); }

/* xoay quanh góc trên-trái thay vì tâm */
.corner {
  transform: rotate(40deg);
  transform-origin: top left;
}

3 Kết hợp nhiều phép & biến đổi 3D Nâng cao

Có thể ghép nhiều hàm trong một transform (thứ tự có ý nghĩa). Thêm perspective ở phần tử cha để tạo chiều sâu, rồi dùng rotateX/rotateY để xoay trong không gian 3D.

Mặt trước
Mặt sau
Thuộc tính cho không gian 3D
Thuộc tính / hàmÝ nghĩa

rotateX(deg) / rotateY(deg)Xoay quanh trục ngang / dọc trong 3D.
rotateZ(deg)Xoay quanh trục Z (tương đương rotate 2D).

/* ghép nhiều hàm — đọc từ trái sang phải, áp dụng phải→trái */
.combo { transform: rotate(15deg) scale(1.1) translateX(10px); }

/* 3D: cha tạo chiều sâu, con xoay */
.scene { perspective: 600px; }
.card  { transform: rotateY(50deg); }

Thứ tự quan trọng! translateX(50px) rotate(45deg) khác rotate(45deg) translateX(50px) — vì sau khi xoay, trục toạ độ của phần tử cũng xoay theo.

4 transition — chuyển trạng thái mượt Cơ bản

transition làm cho việc thay đổi giá trị một thuộc tính diễn ra từ từ thay vì nhảy đột ngột — thường kích hoạt bằng :hover, :focus hay khi đổi class. Gồm 4 thành phần: property · duration · timing-function · delay.

Rê chuột vào ô để xem (phóng to + đổi màu + bo tròn)
Rê chuột
vào tôi
Rê chuột → giãn chữ
Thành phần của transition
Thuộc tínhÝ nghĩaVí dụ
transition-propertyThuộc tính nào sẽ chuyển động (all = tất cả)transform, background-color
transition-durationThời gian (bắt buộc, nếu 0 = không thấy).35s · 300ms
transition-timing-functionKiểu tăng tốc (xem mục 5)ease · linear
transition-delayChờ bao lâu rồi mới bắt đầu.1s
transition (gộp)Viết tắt: property duration timing delaytransform .35s ease .1s
 
/* viết tách từng phần */
.box {
  transition-property: transform, background-color;
  transition-duration: .35s;
  transition-timing-function: ease;
  transition-delay: 0s;
}
/* viết GỘP (shorthand) — khuyên dùng */
.box  { transition: transform .35s ease, background-color .35s ease; }
.box:hover { transform: scale(1.18); background: #04AA6D; }

Bỏ quên duration thì transition không chạy (mặc định là 0s). Luôn ghi thời gian. Và đặt transition ở trạng thái thường, không phải trong :hover, để hiệu ứng mượt cả lúc rê vào lẫn rê ra.

5 timing-function — đường cong tốc độ Nâng cao

Hàm thời gian quyết định tốc độ thay đổi theo thời gian: nhanh dần, chậm dần, đều, giật từng nấc… Rê chuột vào ô bên dưới để cho cả 6 viên chạy cùng lúc, cùng quãng đường, cùng 1.6s và quan sát khác biệt.

Các giá trị timing-function
Giá trịCảm giác chuyển động
ease (mặc định)Chậm → nhanh → chậm; tự nhiên nhất.

ease-inBắt đầu chậm rồi tăng tốc.

.a { transition-timing-function: ease-out; }
.b { transition-timing-function: cubic-bezier(.68,-0.55,.27,1.55); } /* nảy */
.c { transition-timing-function: steps(5, end); }              /* 5 nấc */

Khó nhớ con số cubic-bezier? Mở DevTools của trình duyệt, bấm vào ô timing-function sẽ hiện trình chỉnh đường cong trực quan để kéo thả.

6 @keyframes — định nghĩa các mốc Cơ bản

Khác với transition (chỉ đi từ A → B), @keyframes cho phép định nghĩa nhiều mốc trong một hoạt ảnh: bằng from/to hoặc theo phần trăm 0%…100%. Hai demo dưới đây chạy tự động nên chụp ảnh tĩnh vẫn thấy.

from → to (2 mốc) · tự chạy
from→to
theo % (3 mốc) · tự chạy
0/50/100%
/* Cách 1: chỉ 2 mốc đầu–cuối */
@keyframes grow {
  from { background: #1D559F; transform: scale(1); }
  to   { background: #04AA6D; transform: scale(1.25); }
}

/* Cách 2: nhiều mốc theo phần trăm */
@keyframes rainbow {
  0%   { background: #1D559F; transform: translateY(0); }
  50%  { background: #04AA6D; transform: translateY(-14px); }
  100% { background: #E63946; transform: translateY(0); }
}

from = 0% và to = 100%. Có thể gộp nhiều mốc giống nhau: 0%, 100% { … }. Trong một mốc có thể đổi nhiều thuộc tính cùng lúc.

7 Thuộc tính animation Cơ bản

Sau khi có @keyframes, gắn nó vào phần tử bằng các thuộc tính animation-* (hoặc gộp thành animation). Bảng dưới liệt kê đầy đủ.

Các thuộc tính animation (theo W3Schools)
Thuộc tínhÝ nghĩaGiá trị ví dụ
animation-nameTên @keyframes sẽ chạyspin
animation-durationThời lượng một vòng (bắt buộc > 0)1s · 500ms
animation-timing-functionĐường cong tốc độlinear · ease-in-out
animation-delayChờ bao lâu rồi bắt đầu (âm = bắt đầu giữa chừng).3s
animation-iteration-countSố lần lặp3 · infinite
animation-directionChiều chạy mỗi vòngnormal · reverse · alternate
animation-fill-modeGiữ trạng thái trước/sau khi chạynone · forwards · both
animation-play-stateĐang chạy hay tạm dừngrunning · paused
animation (gộp)Viết tắt tất cả trong một dòngspin 1s linear infinite
 
/* viết tách từng phần */
.loader {
  animation-name: spin;
  animation-duration: 1s;
  animation-timing-function: linear;
  animation-iteration-count: infinite;
}
/* viết GỘP — thứ tự: name duration timing delay count direction fill */
.loader { animation: spin 1s linear infinite; }
.popup  { animation: grow .6s ease-out .2s 1 normal forwards; }

animation-fill-mode: forwards giữ lại trạng thái của mốc 100% sau khi chạy xong (nếu không, phần tử "nhảy" về trạng thái gốc). Dùng both để áp dụng cả mốc đầu trong lúc delay.

8 Ví dụ tự chạy: spinner · pulse · bounce Thực hành

Những hoạt ảnh lặp vô hạn hay gặp nhất: vòng xoay tải trang (spinner), nhịp đập (pulse), quả bóng nảy (bounce), thanh tiến trình. Tất cả tự chạy, không cần tương tác.

Đang chạy tự động (animation: … infinite)
spinner
spinner 2 màu
♥
pulse (nhịp)
bounce (nảy)
progress bar (alternate) 
/* SPINNER — xoay tròn vô hạn */
@keyframes spin { to { transform: rotate(360deg); } }
.spinner {
  width: 56px; height: 56px; border-radius: 50%;
  border: 6px solid #e3e8f0;
  border-top-color: #1D559F;          /* một cạnh khác màu → thấy xoay */
  animation: spin 1s linear infinite;
}

/* PULSE — phình ra + quầng sáng lan toả */
@keyframes pulse {
  0%   { transform: scale(.85); box-shadow: 0 0 0 0 rgba(230,57,70,.45); }
  70%  { transform: scale(1);   box-shadow: 0 0 0 18px rgba(230,57,70,0); }
  100% { transform: scale(.85); }
}

/* BOUNCE — nảy lên xuống */
@keyframes bounce {
  0%, 100% { transform: translateY(0); }
  50%      { transform: translateY(-40px); }
}
.ball { animation: bounce 1s infinite; }

Spinner chỉ cần một cạnh viền khác màu (border-top-color) rồi xoay 360° vô hạn — công thức loader phổ biến nhất, nhẹ và không cần ảnh.

9 Hiệu ứng xuất hiện khi tải trang Thực hành

Cho nội dung hiện dần (fade-in) hoặc trượt vào (slide-in) ngay khi tải. Bí quyết: animation chạy một lần với animation-fill-mode: both, và dùng animation-delay khác nhau để tạo hiệu ứng "đổ xuống lần lượt" (staggered).

Chạy một lần khi trang tải xong
Banner này fade-in (mờ → rõ)
Thẻ 1 — trượt lên, delay .05s
Thẻ 2 — trượt lên, delay .25s
Thẻ 3 — trượt lên, delay .45s
@keyframes slide-up {
  from { opacity: 0; transform: translateY(26px); }
  to   { opacity: 1; transform: translateY(0); }
}
.card { animation: slide-up .7s ease both; } /* both = giữ trạng thái đầu/cuối */
.card:nth-child(1) { animation-delay: .05s; }
.card:nth-child(2) { animation-delay: .25s; }   /* mỗi thẻ chậm hơn một nhịp */
.card:nth-child(3) { animation-delay: .45s; }

Animate opacity + transform (chứ không phải top/margin) để hiệu ứng mượt và không gây "giật layout" — lý do giải thích ở mục 11.

10 Thực hành: nút có hiệu ứng hover Thực hành

Gộp cả ba kỹ thuật: transition để mượt, transform để nhấc nút lên, và box-shadow để tạo cảm giác nổi khối. Rê chuột vào để xem, nhấn giữ để thấy trạng thái :active.

Rê chuột / nhấn giữ vào các nút
.btn {
  color: #fff; padding: 12px 24px; border: none; border-radius: 10px;
  background: linear-gradient(135deg,#1D559F,#6c5ce7);
  box-shadow: 0 6px 14px rgba(29,85,159,.30);
  transition: transform .2s ease, box-shadow .2s ease, filter .2s ease;
}
.btn:hover  { transform: translateY(-4px) scale(1.04); box-shadow: 0 14px 26px rgba(29,85,159,.40); }
.btn:active { transform: translateY(-1px) scale(.99); } /* nhấn xuống */

Quy tắc UI: hover thì "nhấc lên" (bóng đậm hơn, dời lên), active thì "ấn xuống" (bóng nhạt, dời xuống gần vị trí gốc). Tạo phản hồi xúc giác rất tự nhiên.

11 Hiệu năng & tôn trọng người dùng Tổng kết

Hoạt ảnh đẹp nhưng phải mượt và không gây khó chịu. Hai nguyên tắc cốt lõi khi làm hiệu ứng.

1 · Ưu tiên animate transform và opacity. Hai thuộc tính này được trình duyệt tăng tốc bằng GPU và không kích hoạt tính lại layout (reflow). Tránh animate width, height, top, left, margin — chúng làm cả trang phải bố trí lại, gây giật (jank).

2 · Tôn trọng prefers-reduced-motion. Một số người bị chóng mặt với chuyển động. Hãy tắt/giảm hoạt ảnh khi hệ điều hành của họ bật chế độ "giảm chuyển động". Trang này đã áp dụng quy tắc đó cho mọi demo phía trên.

/* Tắt hoạt ảnh cho người chọn "giảm chuyển động" trong hệ điều hành */
@media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}
Nên / tránh khi animate
Mục tiêuNên animateTránh animate
Di chuyểntransform: translate()top / left / margin
Phóng to / thu nhỏtransform: scale()width / height
Hiện / ẩnopacitydisplay (không transition được)

Đừng lạm dụng. Hoạt ảnh phục vụ trải nghiệm, không phải để "khoe". Hiệu ứng quá nhiều/quá dài làm trang nặng và mệt mắt. Mỗi hiệu ứng nên có lý do (phản hồi thao tác, hướng sự chú ý, thể hiện trạng thái tải).
