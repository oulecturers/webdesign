Flexbox cho Responsive

Flexbox là mô hình bố cục một chiều mạnh mẽ nhất để dàn trang co giãn theo màn hình. Từ display:flex cơ bản đến flex-wrap, flex: 1 1 250px — kỹ thuật giúp giao diện tự sắp xếp lại mà cần rất ít media query.

1 display:flex — container, item & hai trục Cơ bản

Đặt display:flex cho một phần tử cha (flex container) thì mọi con trực tiếp của nó trở thành flex item và được xếp theo một đường thẳng. Flexbox là bố cục một chiều: hoặc theo hàng, hoặc theo cột.

Kết quả · cha có display:flex → 3 con xếp thành hàng
item 1
item 2
item 3
/* CHA: chỉ cần 1 dòng để kích hoạt Flexbox */
.container { display: flex; }
/* CON trực tiếp tự động trở thành flex item, không cần khai báo gì thêm */

Trục chính (main axis) — hướng các item được xếp ra, mặc định là ngang (trái → phải). justify-content căn item theo trục này.

Trục phụ (cross axis) — vuông góc với trục chính, mặc định là dọc. align-items căn item theo trục này. Khi đổi flex-direction:column, hai trục hoán đổi vai trò.

2 flex-direction + responsive Cơ bản

flex-direction quyết định hướng của trục chính: row (ngang, mặc định) hay column (dọc). Đây là mẹo responsive kinh điển: màn hình rộng xếp ngang, màn hình hẹp (điện thoại) xếp dọc.

Các giá trị của flex-direction
Giá trịHướng xếp item
row (mặc định)Theo hàng, từ trái sang phải.
row-reverseTheo hàng, từ phải sang trái.
columnTheo cột, từ trên xuống dưới.
column-reverseTheo cột, từ dưới lên trên.

Responsive: row trên màn rộng → column trên màn hẹp

Khối dưới đây xếp ngang bình thường, nhưng khi khung < 560px sẽ tự chuyển sang xếp dọc. Hãy thu hẹp cửa sổ trình duyệt để thấy.

Thu nhỏ trình duyệt < 560px → các khối xếp dọc
Trang chủ
Sản phẩm
Liên hệ
.menu { display: flex;
        flex-direction: row; }     /* mặc định: xếp ngang */

@media (max-width: 560px) {
  .menu { flex-direction: column; } /* màn nhỏ: xếp dọc */
}

Chỉ cần một thuộc tính đổi trong media query là cả khối tự sắp xếp lại — đây là sức mạnh của Flexbox cho responsive: ít CSS, không cần đụng tới HTML.

3 flex-wrap — chìa khoá responsive Cơ bản

Mặc định flex item không xuống hàng (nowrap) — chúng cố ép vào một dòng và bị bóp nhỏ lại. Đặt flex-wrap:wrap cho phép item tự xuống hàng khi không đủ chỗ. Đây chính là cốt lõi của bố cục responsive bằng Flexbox.

Các giá trị của flex-wrap
Giá trịÝ nghĩa
nowrap (mặc định)Tất cả item ép vào một dòng, có thể bị co nhỏ.
wrapItem tràn sẽ xuống hàng mới — nền tảng responsive.
wrap-reverseXuống hàng nhưng các hàng xếp ngược (dưới lên).
 
.grid { display: flex;
        flex-wrap: wrap;   /* hết chỗ thì xuống hàng */
        gap: 16px; }

Viết tắt flex-flow gộp cả hai: flex-flow: row wrap; tương đương flex-direction: row; flex-wrap: wrap;.

6 flex-grow / flex-shrink / flex-basis Nâng cao

Ba thuộc tính đặt trên item điều khiển cách item co giãn để lấp khoảng trống. Viết gọn cả ba bằng shorthand flex.

/* shorthand flex: grow | shrink | basis */
.item-a { flex: 1; }          /* = 1 1 0  → chia đều phần trống */
.item-b { flex: 2 1 0; }      /* lấy gấp đôi phần dư */
.item-c { flex: 0 0 120px; } /* cố định 120px, không co không giãn */
.item-d { flex: 1 1 250px; } /* giãn được, co được, rộng tối thiểu 250px */

flex: 1 1 250px là công thức vàng cho card responsive: mỗi thẻ rộng khoảng 250px, tự giãn lấp chỗ trống, và tự xuống hàng khi hẹp — xem ngay mục kế tiếp.

7 Thực hành: hàng thẻ (cards) responsive Thực hành

Kết hợp flex-wrap:wrap + flex: 1 1 250px tạo một hàng thẻ tự đáp ứng: màn rộng 4 thẻ/hàng, màn vừa 2–3 thẻ, điện thoại 1 thẻ — không cần media query nào.

Thu nhỏ trình duyệt → số thẻ trên mỗi hàng tự thay đổi

Thẻ A

Tự giãn rộng khi còn chỗ, co lại khi hẹp.

Thẻ B

Rộng tối thiểu 250px nhờ flex-basis.

Thẻ C

Xuống hàng tự động khi không đủ chỗ.

Thẻ D

Bố cục responsive chỉ với 3 dòng CSS.

.cards { display: flex;
         flex-wrap: wrap;     /* cho phép xuống hàng */
         gap: 16px; }

.card  { flex: 1 1 250px; }  /* giãn · co · rộng tối thiểu 250px */

Toàn bộ hành vi responsive đến từ một dòng flex: 1 1 250px: basis 250px đặt bề rộng mong muốn, grow 1 cho giãn lấp chỗ, flex-wrap cho xuống hàng. Đổi 250px → 200px sẽ có nhiều cột hơn.

8 Thực hành: navbar responsive Thực hành

Thanh điều hướng là ứng dụng Flexbox phổ biến nhất: logo bên trái, menu bên phải. Dùng margin-right:auto đẩy menu sang phải, và flex-wrap:wrap cho menu tự xuống hàng khi hẹp.

Logo trái · menu phải · tự xuống hàng khi không đủ chỗ
WebDesign Trang chủ Khoá học Bài viết Liên hệ Đăng ký 
.navbar { display: flex;
          flex-wrap: wrap;       /* menu xuống hàng khi hẹp */
          align-items: center;
          gap: 6px 16px; }

.navbar .brand { margin-right: auto; } /* đẩy phần còn lại sang phải */

Mẹo: margin-right:auto (hoặc margin-left:auto) trên một item "ăn" hết khoảng trống còn lại, đẩy các item sau dạt về phía đối diện — cực tiện để tách logo và menu.

9 Căn giữa hoàn hảo theo cả 2 chiều Nâng cao

Bài toán kinh điển "căn giữa một hộp cả ngang lẫn dọc" từng rất khó, nay chỉ cần 2 dòng với Flexbox: justify-content:center (ngang) + align-items:center (dọc).

Hộp xanh nằm chính giữa theo cả hai chiều
Căn giữa
hoàn hảo ✓
.parent {
  display: flex;
  justify-content: center; /* căn giữa theo trục chính (ngang) */
  align-items: center;     /* căn giữa theo trục phụ (dọc) */
  min-height: 160px;        /* cần có chiều cao để căn dọc thấy rõ */
}

Để căn giữa giữa màn hình, đặt container cao bằng cả viewport: min-height: 100vh rồi cũng justify-content:center; align-items:center; — mọi nội dung bên trong sẽ nằm chính giữa trang.

11 Mẹo responsive với Flexbox Tổng kết

Mẹo 1 — Thay nhiều media query. Cặp flex-wrap:wrap + flex-basis (hoặc flex: 1 1 250px) tự cho item xuống hàng theo bề rộng màn hình. Một dòng này thay được hàng loạt breakpoint thủ công.

Mẹo 2 — Tránh item bị bẹp. Khi nowrap, item có thể bị co tới mức méo nội dung. Đặt min-width (hoặc flex-basis đủ lớn) để giữ kích thước tối thiểu, hoặc bật flex-wrap:wrap cho xuống hàng.

Mẹo 3 — row → column trên mobile. Trong media query nhỏ, chỉ cần flex-direction: column là biến menu/khối ngang thành dọc cho dễ chạm trên điện thoại.

Lưu ý — Flexbox là 1 chiều. Cần dóng hàng và cột thành lưới đều thật sự 2 chiều thì Grid phù hợp hơn (xem bài tiếp). Flexbox mạnh nhất cho navbar, hàng thẻ, căn giữa, và sắp xếp theo một trục.
