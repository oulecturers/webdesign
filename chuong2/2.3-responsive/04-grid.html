CSS Grid cho thiết kế Responsive

Grid là hệ thống dàn trang 2 chiều (hàng × cột) mạnh nhất của CSS. Từ chia cột bằng fr, repeat(), đến lưới tự co giãn với auto-fit + minmax() (không cần media query) và bố cục cả trang bằng grid-template-areas.

1 display:grid — container & item Cơ bản

Đặt display:grid cho một phần tử cha → nó trở thành grid container; mọi con trực tiếp tự động thành grid item và được xếp vào các ô của lưới. Lưới là khung 2 chiều gồm hàng (row) và cột (column).

Kết quả · 6 item trong lưới 3 cột
01
02
03
04
05
06
/* CHA: bật chế độ lưới */
.container { display: grid;
             grid-template-columns: 1fr 1fr 1fr; /* 3 cột bằng nhau */
             gap: 10px; }
/* Mọi <div> con trực tiếp tự thành grid item, tự xuống hàng khi đủ 3 cột */

Chỉ con trực tiếp mới là grid item (cháu thì không). Item tự động chảy từ trái sang phải, hết một hàng thì sang hàng mới — không cần đếm trước số phần tử.

Còn display:inline-grid giống grid nhưng bản thân container lại nội tuyến.

Grid vs Flexbox: Flexbox dàn theo một chiều (một hàng hoặc một cột); Grid dàn theo hai chiều (hàng và cột cùng lúc) — lý tưởng cho bố cục cả trang.

2 grid-template-columns & đơn vị fr Cơ bản

grid-template-columns khai báo số cột và bề rộng từng cột. Đơn vị fr (fraction — phần) chia không gian còn lại theo tỉ lệ, giúp lưới co giãn mềm mại.

Kết quả · tỉ lệ 1fr : 2fr : 1fr (cột giữa rộng gấp đôi)
1fr
2fr
1fr
150px (cố định)
1fr (phần còn lại)
/* 3 cột tỉ lệ 1 : 2 : 1 — cột giữa rộng gấp đôi hai cột bên */
.a { grid-template-columns: 1fr 2fr 1fr; }

/* trộn đơn vị: cột trái cố định 150px, cột phải ăn hết phần còn lại */
.b { grid-template-columns: 150px 1fr; }

/* dùng px / % / fr / auto đều được */
.c { grid-template-columns: 200px 25% 1fr auto; }
Đơn vị thường dùng cho kích thước cột/hàng
Đơn vịÝ nghĩaVí dụ
frMột phần của không gian còn lại (linh hoạt, responsive).1fr 2fr
pxBề rộng cố định, không co giãn.200px
%Phần trăm bề rộng của container.25%
autoTự khít theo nội dung của cột.auto

Dùng fr thay cho % khi muốn lưới tự cân mà không phải tự trừ phần gap — fr đã tính khoảng cách giúp bạn rồi.

3 repeat() — rút gọn lặp cột Cơ bản

Khi nhiều cột giống nhau, gõ 1fr 1fr 1fr 1fr rất dài. Hàm repeat(số_lần, kích_thước) viết gọn lại. Có thể trộn với cột thường.

Kết quả · repeat(4, 1fr) và repeat(2, 1fr) 2fr
01
02
03
04
1fr
1fr
2fr (rộng hơn)
/* Hai cách viết tương đương: */
.a { grid-template-columns: 1fr 1fr 1fr 1fr; }
.b { grid-template-columns: repeat(4, 1fr); }   /* gọn hơn ✓ */

/* Trộn: hai cột đầu 1fr, cột cuối 2fr */
.c { grid-template-columns: repeat(2, 1fr) 2fr; }

/* Lặp một cụm mẫu: 1fr 2fr lặp lại 2 lần → 4 cột */
.d { grid-template-columns: repeat(2, 1fr 2fr); }

repeat() dùng được cho cả grid-template-rows. Đặc biệt, kết hợp với auto-fit/auto-fill ở mục sau sẽ tạo lưới responsive tự động — phần quan trọng nhất của bài.

4 minmax() + auto-fit — lưới responsive Quan trọng

Đây là kỹ thuật responsive đắt giá nhất của Grid: repeat(auto-fit, minmax(200px, 1fr)) tạo lưới tự đổi số cột theo bề rộng màn hình — KHÔNG cần một dòng media query nào. Thử thu/phóng cửa sổ để thấy số cột thay đổi.

minmax(min, max) = mỗi cột rộng tối thiểu min và tối đa max. Ở đây minmax(200px, 1fr): mỗi cột không hẹp hơn 200px, nhưng được phép giãn ra chia đều (1fr).

auto-fit = trình duyệt tự tính nhét được bao nhiêu cột 200px trên hàng rồi giãn chúng lấp đầy. Màn rộng → nhiều cột; màn hẹp → ít cột; rất hẹp → 1 cột.

   nhiều nhất chia đều 1fr". Trình duyệt tự tăng/giảm số cột theo bề rộng. */
auto-fit khác auto-fill thế nào?
Từ khoáKhi còn dư chỗ (ít item, màn rộng)
auto-fitCo các cột rỗng về 0 → các item hiện có giãn ra lấp đầy hàng.
auto-fillGiữ các cột rỗng (kích thước min) → item không giãn, để lại khoảng trống bên phải.

Mặc định nên dùng auto-fit cho gallery/card để các phần tử luôn lấp đầy hàng đẹp mắt. Chỉ dùng auto-fill khi muốn giữ nguyên "ô" cho dù chưa có nội dung.

5 gap — khoảng cách giữa các ô Cơ bản

gap tạo rãnh giữa các hàng và cột mà không sinh khoảng thừa ở mép ngoài (khác với dùng margin). Có thể tách riêng row-gap và column-gap.

Kết quả · row-gap:24px (dọc lớn) · column-gap:6px (ngang nhỏ)
01
02
03
04
05
06
.a { gap: 10px; }            /* hàng & cột đều 10px */
.b { gap: 24px 6px; }         /* row-gap 24px, column-gap 6px */
.c { row-gap: 24px;
     column-gap: 6px; }        /* viết tách riêng cũng được */

gap còn dùng được cho cả Flexbox. Trước đây nó tên là grid-gap; nay gap là chuẩn — cứ dùng gap.

6 grid-template-rows & lưới ngầm (auto-rows) Nâng cao

grid-template-rows đặt chiều cao cho các hàng tường minh. Khi item nhiều hơn số hàng đã khai báo, Grid tự sinh thêm hàng ngầm (implicit); chiều cao của chúng do grid-auto-rows quyết định.

Kết quả · 2 hàng tường minh (80px, 40px) + hàng ngầm 60px
80px
80px
80px
40px
40px
40px
ngầm 60px
ngầm 60px
.grid {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: 80px 40px;   /* chỉ định 2 hàng đầu */
  grid-auto-rows: 60px;          /* mọi hàng phát sinh thêm cao 60px */
}

Để các hàng tự cao theo nội dung mà vẫn có chiều cao tối thiểu, dùng grid-auto-rows: minmax(60px, auto).

7 grid-template-areas — vẽ bố cục trang Nâng cao

Cách trực quan nhất để dàn cả trang: đặt tên cho từng vùng rồi "vẽ" bố cục bằng chữ. Mỗi chuỗi là một hàng; mỗi tên là một ô; tên lặp lại = vùng trải rộng nhiều ô.

Kết quả · header trải ngang · sidebar | main · footer trải ngang
Header
Sidebar
Main content
Footer
.page {
  display: grid;
  grid-template-areas:
    "header  header"     /* hàng 1: header trải 2 cột */
    "sidebar main"       /* hàng 2: sidebar | main   */
    "footer  footer";    /* hàng 3: footer trải 2 cột */
  grid-template-columns: 200px 1fr;
  grid-template-rows: 56px 1fr 44px;
}
/* Gán mỗi phần tử vào vùng theo TÊN */
.page .header  { grid-area: header; }
.page .sidebar { grid-area: sidebar; }
.page .main    { grid-area: main; }
.page .footer  { grid-area: footer; }

Số tên trên mỗi hàng phải bằng số cột; số hàng phải bằng số dòng chuỗi. Dùng dấu chấm "." để chừa một ô trống. Đây là cách dễ đọc nhất để mô tả layout — nhìn CSS là thấy ngay bố cục.

8 Đổi bố cục areas theo @media Responsive

Ưu điểm lớn của grid-template-areas: muốn responsive chỉ cần vẽ lại sơ đồ vùng trong media query — không phải đụng tới HTML. Dưới đây màn rộng xếp 3 cột, màn hẹp (≤600px) dồn về 1 cột xếp dọc.

Kết quả · thu cửa sổ ≤ 600px để thấy dồn 1 cột
Nav
Sidebar
Main
Footer
/* Mặc định (màn rộng): nav trên · sidebar | main · footer */
.page {
  grid-template-areas:
    "nav  nav  nav"
    "side main main"
    "foot foot foot";
  grid-template-columns: 1fr 1fr 1fr;
}

/* Màn hẹp: vẽ lại sơ đồ → tất cả xếp DỌC trên 1 cột */
@media (max-width: 600px) {
  .page {
    grid-template-areas:
      "nav"
      "main"
      "side"
      "foot";
    grid-template-columns: 1fr;
  }
}

Để ý thứ tự trong mobile: main được đưa lên trên side dù trong HTML sidebar đứng trước — Grid cho phép đổi thứ tự hiển thị mà không sửa cấu trúc HTML, rất tiện cho responsive.

9 Đặt vị trí item: grid-column / grid-row / span Nâng cao

Mỗi item có thể được "ghim" vào vị trí cụ thể hoặc cho chiếm nhiều ô. Lưới đánh số đường kẻ (line) từ 1; grid-column: 1 / 3 nghĩa là từ đường 1 đến đường 3 (chiếm 2 cột). Hoặc dùng span N để nói "chiếm N ô" mà không cần đếm đường.

Kết quả · ô hồng chiếm 2 cột, ô xanh chiếm 2 hàng, ô cam trải cả hàng
A · 2 cột
B · 2 hàng
C
D
E
F
G · span 4 (cả hàng)
/* Cách 1: chỉ định đường kẻ bắt đầu / kết thúc */
.big  { grid-column: 1 / 3; }   /* từ đường 1 → 3 = chiếm 2 cột */

/* Cách 2: dùng span — gọn, không phải đếm đường */
.tall { grid-row: span 2; }   /* chiếm 2 hàng */
.wide { grid-column: span 4; } /* trải hết 4 cột */

/* Viết gộp cả hàng & cột bằng grid-area: row-start / col-start / row-end / col-end */
.box  { grid-area: 1 / 1 / 3 / 3; } /* ô 2×2 ở góc trên-trái */

Dùng số âm để đếm ngược từ mép phải: grid-column: 1 / -1 nghĩa là "trải từ cột đầu đến cột cuối cùng" — mẹo hay để một phần tử chiếm trọn chiều ngang lưới.

10 Thuộc tính của container vs item Tra cứu

Ghi nhớ then chốt: thuộc tính nào đặt trên cha (container), thuộc tính nào đặt trên con (item).

11 Mẹo & ghi nhớ Tổng kết

Gallery responsive. Cần lưới ảnh/card tự đổi số cột theo màn hình mà không viết media query? Một dòng là đủ: grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)).

Layout cả trang. Header / Sidebar / Main / Footer → dùng grid-template-areas; muốn responsive thì vẽ lại sơ đồ vùng trong @media, không phải sửa HTML.

Flex hay Grid? Một dãy phần tử theo 1 chiều (navbar, hàng nút) → flex. Khung 2 chiều hàng × cột (bố cục trang, gallery) → grid. Hai cái thường dùng chung trong một trang.

Lưu ý. Chỉ con trực tiếp mới là grid item. Nếu phần tử cần dàn nằm sâu hơn một cấp, lưới sẽ không "thấy" chúng — hãy bọc đúng cấp hoặc cho lớp trung gian cũng là grid.

Dễ nhầm. 1fr chia không gian còn lại chứ không phải toàn bộ; nếu trong lưới có cột px cố định, các cột fr chỉ chia phần dư sau khi trừ cột cố định và gap.
