2 Giải mẫu 1 — Tính chỉ số BMI

Lấy giá trị bằng parseFloat, validate trước khi tính, phân loại bằng chuỗi if, xuất bằng textContent (an toàn). Hàm phân loại tách riêng — hàm thuần đúng chuẩn bài 3.5.

Chạy thử — BMI = kg / (m × m)
// thử 60kg · 1.65m — hoặc bỏ trống để xem validate
const phanLoai = (bmi) => {          // hàm THUẦN: nhận số, trả chuỗi
  if (bmi < 18.5) return "Thiếu cân";
  if (bmi < 25)   return "Bình thường";
  if (bmi < 30)   return "Thừa cân";
  return "Béo phì";
};

btn.addEventListener("click", () => {
  const kg = parseFloat(oKg.value);
  const m  = parseFloat(oM.value);
  if (isNaN(kg) || isNaN(m) || kg <= 0 || m <= 0) {   // validate
    out.textContent = "Vui lòng nhập cân nặng và chiều cao hợp lệ (> 0).";
    return;                                            // thoát sớm
  }
  const bmi = kg / (m * m);
  out.textContent = `BMI = ${bmi.toFixed(1)} → ${phanLoai(bmi)}`;
});

3 Giải mẫu 2 — Đổi tiền tệ

Tỉ giá lưu trong object tra cứu (key = mã tiền). Ô rỗng / không hợp lệ → bật class loi (viền đỏ nhấp nháy — trình bày nằm trong CSS, đúng nguyên tắc 3 tầng); gõ lại thì tự gỡ lỗi.

Chạy thử — VNĐ → USD / EUR / AUD
// bấm Đổi khi ô rỗng để xem viền đỏ nhấp nháy
/* CSS của hiệu ứng lỗi: */
.loi { border: 2px solid red; animation: nhapnhay .6s ease-in-out 2; }
@keyframes nhapnhay { 50% { opacity: .3; } }

// 1 VNĐ = ... (tỉ giá minh hoạ)
const tiGia = { USD: 1 / 25000, EUR: 1 / 27000, AUD: 1 / 16500 };

btn.addEventListener("click", () => {
  const vnd = parseFloat(oVnd.value);
  if (oVnd.value.trim() === "" || isNaN(vnd) || vnd < 0) {
    oVnd.classList.add("loi");                 // bật viền đỏ
    kq.textContent = "Vui lòng nhập số tiền hợp lệ.";
    return;
  }
  oVnd.classList.remove("loi");
  const ma = document.getElementById("loaiTien").value;
  kq.textContent = `${vnd.toLocaleString("vi-VN")} VNĐ = ${(vnd * tiGia[ma]).toFixed(2)} ${ma}`;
});

oVnd.addEventListener("input", () => oVnd.classList.remove("loi"));

4 Giải mẫu 3 — To-Do List + localStorage Nâng cao

Đủ combo kỹ thuật chương 3: createElement + textContent (chống XSS), event delegation (một listener cho cả danh sách), Enter để thêm nhanh, confirm khi xoá — và bản nâng cao của giáo trình: lưu localStorage, F5 không mất việc.

Chạy thử — thêm việc, đánh dấu ✓, xoá ✕, rồi F5 xem còn không
    // dữ liệu đồng bộ vào localStorage["bt-todos"] sau mỗi thao tác
    // ---- dữ liệu là MẢNG OBJECT, DOM chỉ là "ảnh chụp" của dữ liệu ----
let todos = JSON.parse(localStorage.getItem("bt-todos")) || [];

const luu = () => localStorage.setItem("bt-todos", JSON.stringify(todos));

function render() {                       // vẽ lại toàn bộ danh sách từ mảng
  ds.innerHTML = "";
  todos.forEach((viec) => {
    const li = document.createElement("li");
    li.innerHTML = `<span></span><button class="ok">✓</button><button class="del">✕</button>`;
    li.querySelector("span").textContent = viec.noiDung;  // chống XSS
    li.querySelector("span").classList.toggle("xong", viec.xong);
    ds.appendChild(li);
  });
}

// event delegation: MỘT listener xử lý mọi nút ✓ / ✕
ds.addEventListener("click", (e) => {
  const li = e.target.closest("li");
  if (!li) return;
  const i = [...ds.children].indexOf(li);     // vị trí trong mảng
  if (e.target.classList.contains("ok"))  todos[i].xong = !todos[i].xong;
  if (e.target.classList.contains("del") && confirm("Xoá việc này?"))
    todos.splice(i, 1);
  luu(); render();                              // mọi thay đổi: lưu + vẽ lại
});

    Tư duy đáng giá nhất bài này: thao tác trên mảng dữ liệu rồi render lại, thay vì "mò mẫm" sửa từng thẻ DOM. Đây chính là tư duy state → view của React/Vue sau này (bài 3.13).

    6 Mẹo làm bài

    Quy trình 4 bước cho mọi bài: ① dựng HTML tối thiểu → ② viết hàm thuần xử lý (tính toán, không đụng DOM) → ③ nối vào giao diện bằng addEventListener → ④ validate + thông báo lỗi đẹp.

    Ba lỗi chấm bài gặp nhiều nhất: quên parseFloat với input.value (cộng chuỗi!); script đặt ở <head> không defer (null); gắn sự kiện từng phần tử động thay vì delegation.

    Dùng AI có trách nhiệm (CLO4): được hỏi AI để giải thích, gợi ý hướng, review code; không nộp code không hiểu — bạn phải chịu trách nhiệm từng dòng mình giao.
