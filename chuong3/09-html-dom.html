1 DOM là gì? Cơ bản

Khi trang được nạp, trình duyệt dựng Document Object Model — cây đối tượng trong đó mỗi thẻ HTML là một node. JS dùng cây này để đọc/sửa nội dung, cấu trúc và kiểu dáng trang.

document                      ← gốc của cả cây
 └─ html
     ├─ head ── title, meta…
     └─ body
         ├─ h1
         ├─ p ── "đoạn văn" (text node)
         └─ ul ── li, li, li

DOM định nghĩa: các element (mỗi thẻ là một object), các attribute, các method để thao tác, và các event gắn trên phần tử — trọn bộ "API của trang web".

// querySelector dùng CÚ PHÁP CSS SELECTOR — kiến thức Chương 2 tái sử dụng 100%:
document.querySelector("#menu");          // theo id
document.querySelector(".card");          // theo class (phần tử ĐẦU tiên)
document.querySelectorAll("ul.tasks li"); // tất cả li trong ul.tasks
document.querySelectorAll("input[type=checkbox]:checked"); // selector phức tạp vô tư

Chuẩn nghề: ưu tiên querySelector / querySelectorAll — một cú pháp thống nhất, chọn được mọi thứ. Cần phần tử thuộc nhiều thẻ & class khác nhau → querySelectorAll (câu trắc nghiệm 14).

4 Đổi CSS: style & classList Cơ bản

// Cách 1: sửa style trực tiếp — chú ý tên camelCase!
e.style.backgroundColor = "gold";   // CSS: background-color

// Cách 2 (hiện đại): đổi CLASS — trình bày để yên trong file CSS
e.classList.add("noi-bat");
e.classList.remove("noi-bat");
e.classList.toggle("dark-mode");    // có thì gỡ, chưa có thì thêm
e.classList.contains("dark-mode");  // true/false
Thử ngay — Dark Mode bằng classList.toggle

Góc Cà Phê

JS chỉ toggle class dark-mode — toàn bộ màu sắc (nền, chữ, tiêu đề) và cả transition mượt đều nằm trong CSS.

Chuẩn nghề: ưu tiên đổi class thay vì sửa từng e.style.* — trình bày ở CSS, JS chỉ bật/tắt trạng thái: đúng nguyên tắc tách 3 tầng. Chỉ sửa style trực tiếp khi giá trị phải tính toán động (vị trí px, kích thước…).

children vs childNodes (câu trắc nghiệm 19): children chỉ gồm phần tử con; childNodes gồm tất cả node con — kể cả text node (khoảng trắng xuống dòng giữa các thẻ!) và comment. Duyệt DOM nên dùng children để khỏi dính node "vô hình".

// trong handler: từ nút bấm lần sang span đứng kề trước nó
button.addEventListener("click", function () {
  this.previousElementSibling.classList.toggle("delete");
});  // ⚠ cần this = nút bấm → PHẢI dùng function thường (không phải arrow)

Event delegation — uỷ quyền sự kiện Kỹ thuật quan trọng

Gắn một listener ở phần tử cha rồi kiểm tra event.target — nhờ bubbling, click ở mọi con (kể cả con thêm sau này) đều "nổi" lên cha.

let list = document.querySelector("ul.tasks");
list.addEventListener("click", (event) => {
  if (event.target.tagName === "BUTTON") {      // chỉ xử lý khi click nút xoá
    if (confirm("Bạn chắc chắn xoá?"))
      list.removeChild(event.target.parentElement);
  }
});

11 Lazy loading Nâng cao

Trì hoãn tải ảnh/video/iframe đến khi người dùng cuộn gần tới — tăng tốc nạp trang, tiết kiệm băng thông (liên hệ Core Web Vitals — Chương 1).

<!-- HTML5 có sẵn — một thuộc tính là xong: -->
<img src="image.jpg" alt="…" loading="lazy">
<video preload="none" poster="thumb.jpg" controls>…</video>

// kiểm soát chi tiết hơn: IntersectionObserver — theo dõi phần tử vào viewport
const io = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting) {
      e.target.src = e.target.dataset.src;   // lúc này mới gán src thật
      io.unobserve(e.target);
    }
  });
});
document.querySelectorAll("img[data-src]").forEach(img => io.observe(img));

12 Mẹo & lỗi thường gặp Tổng kết

Lỗi 1. Gán dữ liệu người dùng bằng innerHTML → lỗ hổng XSS. Dữ liệu ngoài → textContent.

Lỗi 2. Gắn listener cho từng phần tử con rồi thêm phần tử mới thì "chết" nút — dùng event delegation ở cha.

Lỗi 3. e.style.background-color — sai; trong JS phải camelCase: backgroundColor (câu trắc nghiệm 12).

Lỗi 4. Bấm nút "chạy interval" nhiều lần không clear → hiệu ứng nhanh dần bất thường vì nhiều interval chồng nhau.

Ghi nhớ. Chọn: querySelector · Nội dung: textContent (an toàn) · Kiểu dáng: classList · Danh sách động: delegation · Hẹn giờ: setTimeout/setInterval + clear.
