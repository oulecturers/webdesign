1 Canvas là gì? · context Cơ bản

<canvas> (HTML5) là vùng vẽ trống — mọi nét vẽ đều do JavaScript ra lệnh qua đối tượng context.

<canvas id="myCanvas" width="500" height="300"></canvas>

const ctx = document.getElementById("myCanvas").getContext("2d");
// "2d" cho đồ hoạ phẳng; "webgl" cho 3D

Đặt kích thước canvas bằng thuộc tính width/height (như trên), không bằng CSS — CSS chỉ kéo giãn tấm vẽ làm hình bị nhoè/méo.

3 Bóng nảy — Play / Stop Nâng cao

Nguyên lý mọi animation canvas: mỗi "khung hình" = xoá → tính toạ độ mới → vẽ lại, lặp bằng setInterval (bài tập 6 dùng đúng kỹ thuật này).

let x = 50, y = 50, dx = 3, dy = 2;              // vị trí + vận tốc

function khungHinh() {
  ctx.clearRect(0, 0, W, H);                     // 1. xoá khung cũ
  x += dx;  y += dy;                             // 2. di chuyển
  if (x < r || x > W - r) dx = -dx;              // 3. chạm biên → đảo hướng
  if (y < r || y > H - r) dy = -dy;
  ctx.beginPath(); ctx.arc(x, y, r, 0, 2 * Math.PI); ctx.fill();  // 4. vẽ lại
}
let id = setInterval(khungHinh, 16);             // ~60 khung/giây · Stop: clearInterval(id)
Thử ngay — quả bóng nảy trong khung

4 SVG là gì? Cơ bản

Scalable Vector Graphics — đồ hoạ vector mô tả bằng thẻ kiểu XML: phóng to không vỡ nét, nhẹ, và vì mỗi hình là một phần tử DOM nên style/gắn sự kiện được như thẻ HTML thường.

  • Nhúng thẳng vào HTML bằng thẻ <svg>, hoặc qua <img src="hinh.svg">.
  • Thuộc tính chung: fill (màu nền), stroke (viền), stroke-width, opacity, transform.
  • viewBox định nghĩa "khung nhìn" — bí quyết để SVG co giãn responsive.

6 SVG tương tác Nâng cao

Vì mỗi hình SVG là phần tử DOM, ta querySelector + addEventListener + đổi thuộc tính như thường — nền tảng của bài tập 10 (chọn hình, tô màu).

const tron = document.querySelector("#hinh-tron");
tron.addEventListener("click", () => {
  tron.setAttribute("fill", mauNgauNhien());   // đổi thuộc tính SVG = setAttribute
});

8 Mẹo & lỗi thường gặp Tổng kết

Lỗi 1. Vẽ hình tròn canvas quên beginPath() → các nét dính vào nhau thành "mạng nhện".

Lỗi 2. Animation quên clearRect đầu mỗi khung → bóng để lại "vệt" dài.

Lỗi 3. Đổi màu SVG bằng e.style.fill có lúc không ăn thuộc tính fill="" — dùng setAttribute("fill", ...) cho chắc.

Ghi nhớ. Icon/logo/biểu đồ → SVG; game/hiệu ứng nhiều pixel → Canvas. Canvas animation = xoá → tính → vẽ lại mỗi khung.
