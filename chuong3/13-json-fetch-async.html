1 Web động & API là gì? Cơ bản

API (Application Programming Interface) phía web thường là một địa chỉ URL trả về dữ liệu (thay vì trang HTML). Trình tự quen thuộc:

Trang web ──fetch(url)──▶ Máy chủ API
Trang web ◀──JSON──────── Máy chủ API
JS parse JSON → render ra DOM (map + template literal)

Ví dụ ứng dụng trong dự án "Góc Cà Phê": danh sách món không viết cứng trong HTML mà nạp từ menu.json — sửa menu chỉ cần sửa file dữ liệu, không đụng vào code.

2 JSON: parse · stringify Cơ bản

JSON (JavaScript Object Notation) — định dạng trao đổi dữ liệu phổ biến nhất, trông gần giống object JS nhưng là chuỗi văn bản (key bắt buộc trong nháy kép).

let obj  = JSON.parse(text);       // chuỗi JSON → object JS (để dùng)
let text = JSON.stringify(obj);    // object JS → chuỗi JSON (để gửi/lưu)

3 Bất đồng bộ là gì? Cơ bản

Tải dữ liệu mất thời gian (mạng chậm, server xa). Đồng bộ = đứng chờ, trang "đơ". Bất đồng bộ = đặt việc chạy nền, trang vẫn mượt, xong việc thì báo.

Thử ngay — thứ tự chạy bất đồng bộ khiến người mới bối rối
// đố bạn: log "1", setTimeout(log "2", 0), log "3" — in ra theo thứ tự nào?

Kết quả 1 → 3 → 2: dù hẹn 0ms, callback của setTimeout vẫn phải xếp hàng đợi code chính chạy xong. Đó là cách JavaScript (đơn luồng) xử lý việc chờ đợi — và là lý do cần Promise/async-await để "viết code chờ" cho dễ đọc.

4 callback → Promise → async/await Nâng cao

// ① CALLBACK (cũ): lồng nhau nhiều tầng → "callback hell"
taiMenu(function (menu) {
  taiGia(menu, function (gia) {
    taiKhuyenMai(gia, function (km) { /* ... lồng nữa ... */ });
  });
});

// ② PROMISE (ES6): phẳng hơn, nối bằng .then()
fetch("menu.json")
  .then(res => res.json())
  .then(data => render(data))
  .catch(err => console.error(err));

// ③ ASYNC/AWAIT (ES2017): đọc như code thường — CHUẨN HIỆN NAY ✓
async function loadMenu() {
  const res  = await fetch("menu.json");
  const data = await res.json();
  render(data);
}

Tự kiểm tra (giáo trình): vì sao dùng async/await thay vì lồng callback? — Tránh "callback hell": code phẳng, đọc từ trên xuống như code đồng bộ, và bắt lỗi gọn bằng try/catch.

5 fetch + async/await + try/catch Trọng tâm

async function loadMenu() {
  try {
    const res = await fetch("menu.json");       // 1. chờ phản hồi
    if (!res.ok) throw new Error(res.status);    // 2. kiểm tra mã trạng thái (404, 500…)
    const data = await res.json();               // 3. đọc & parse JSON
    render(data);                                 // 4. hiển thị ra DOM
  } catch (err) {
    console.error("Lỗi tải menu:", err);         // mạng đứt / file không có / JSON hỏng
  }
}

    7 POST · HTTP status · CORS Nâng cao

    // GET (mặc định): LẤY dữ liệu → fetch(url)

const res = await fetch(url, {
  method: "POST",

    res.oktrue khi status 200–299; fetch không tự ném lỗi khi 404/500 — phải tự kiểm tra!
    Status quen thuộc200 OK · 404 Not Found · 500 Server Error (đã học Chương 1)
    CORSCơ chế bảo mật: trình duyệt chặn gọi API khác domain nếu server không cấp phép. Gặp lỗi CORS khi học là chuyện thường — hiểu rằng đó là server chưa cho phép, không phải code sai cú pháp.

    8 An toàn: API key CLO4

    Không bao giờ để lộ API key/secret trong JS phía client — ai mở DevTools cũng đọc được (View Source, tab Network). Khoá bí mật phải nằm ở back-end; client chỉ gọi qua server trung gian của mình. Kết hợp cảnh báo XSS (3.8) và localStorage (3.9): mọi thứ ở client đều công khai.

    9 Mẹo & lỗi thường gặp Tổng kết

    Lỗi 1. Quên await trước fetch/res.json() → nhận về Promise {<pending>} thay vì dữ liệu.

    Lỗi 2. Tưởng fetch tự báo lỗi khi 404 — không! Phải kiểm tra res.ok rồi throw.

    Lỗi 3. await chỉ dùng được trong hàm có chữ async (hoặc module) — quên là SyntaxError ngay.

    Mẹo. Học/demo fetch file cục bộ → luôn chạy qua Live Server. Xem request thật trong DevTools → tab Network.

    Ghi nhớ. JSON = chuỗi (parse/stringify) · async/await + try/catch là chuẩn viết · kiểm tra res.ok · CORS là bảo mật trình duyệt · key bí mật ở back-end.
