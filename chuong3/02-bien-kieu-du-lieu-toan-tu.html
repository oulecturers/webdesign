1 Kiểu động & typeof Cơ bản

Cùng một biến có thể lần lượt chứa số, chuỗi, boolean… Kiểm tra kiểu hiện tại bằng toán tử typeof.

let x = 5;          // typeof x → "number"
x = "năm";           // typeof x → "string"  (cùng biến, kiểu đã đổi!)
x = true;            // typeof x → "boolean"
Thử ngay — máy soi kiểu dữ liệu

Gõ một biểu thức bất kỳ (ví dụ 5, "5", true, [1,2], {a:1}, null, 1/0):

// kết quả hiện ở đây

Hai kết quả "kỳ quặc" nổi tiếng: typeof null → "object" (lỗi lịch sử của JS, được giữ vĩnh viễn) và typeof NaN → "number" (NaN = "Not a Number" nhưng vẫn thuộc kiểu number!).

2 Kiểu nguyên thuỷ (primitive) Cơ bản

Lưu trực tiếp giá trị — gán sang biến khác là sao chép giá trị.

KiểuÝ nghĩaVí dụ
StringChuỗi ký tự"Hà Nội", 'xin chào'
NumberSố nguyên lẫn số thực (chung một kiểu)42, 3.14, NaN, Infinity
BooleanĐúng / saitrue, false
UndefinedĐã khai báo, chưa có giá trịlet a; // a === undefined
nullRỗng có chủ đích (người viết tự gán)let b = null;
Symbol ES6Giá trị duy nhất, không trùng lặpSymbol("id")
BigInt ES2020Số nguyên rất lớn9007199254740993n

undefined vs null: undefined là "chưa kịp có gì" (JS tự gán); null là "cố ý để trống" (người viết gán). So sánh: null == undefined → true nhưng null === undefined → false.

3 Kiểu tham chiếu (reference) Nâng cao

Object, Array, Function, Date, Map, Set… lưu địa chỉ tham chiếu — gán sang biến khác là sao chép địa chỉ, hai biến cùng trỏ một chỗ.

// PRIMITIVE — sao chép GIÁ TRỊ: b nhận bản sao, sửa b không đụng a
let a = 10;
let b = a;
b = 99;
console.log(a);        // 10  (a không đổi)

// REFERENCE — sao chép ĐỊA CHỈ: sv2 và sv1 cùng trỏ MỘT object
let sv1 = { ten: "An", tuoi: 20 };
let sv2 = sv1;
sv2.tuoi = 99;
console.log(sv1.tuoi); // 99  (!!! sv1 cũng đổi theo)
Thử ngay — bẫy sao chép object
// bấm để xem primitive và object khác nhau thế nào

Muốn sao chép object thật sự? Dùng spread: let sv2 = { ...sv1 }; (bản sao nông — học kỹ ở bài 3.6). Đây là nguồn gốc của rất nhiều lỗi khó hiểu ở người mới — hãy nhớ demo trên.

4 Biến: var / let / const Cơ bản

Ba từ khoá khai báo biến — nhưng code hiện đại chỉ dùng hai.

Từ khoáPhạm viGán lại?Khuyến nghị
varTheo hàm (function scope), bị hoistingĐược✗ Không dùng trong code mới
letTheo khối { } (block scope)Được✓ Khi cần gán lại
constTheo khối { }Không✓ Mặc định dùng const

Quy tắc đặt tên

  • Phân biệt hoa/thường; không trùng từ khoá (var, function, break…).
  • Bắt đầu bằng chữ cái, _ hoặc $ — không bắt đầu bằng chữ số.
  • Quy ước camelCase: firstName, totalPrice, giaSanPham.

const với object/array: không gán lại được biến, nhưng vẫn sửa được nội dung bên trong: const a = [1]; a.push(2); ✓ — còn a = [9]; ✗ TypeError. const khoá "địa chỉ", không khoá "ruột".

5 Phạm vi (scope) Nâng cao

Biến "sống" ở đâu thì chỉ dùng được ở đó. var rò rỉ ra ngoài khối — lý do chính khiến nó bị khai tử.

function testScope() {
  if (true) {
    var a = 1;         // var: phạm vi THEO HÀM
    let b = 2;         // let: phạm vi THEO KHỐI { }
    const c = 3;
  }
  console.log(a);        // 1   → var "rò" ra ngoài khối if!
  console.log(b);        // ✗ ReferenceError: b is not defined
  console.log(c);        // ✗ ReferenceError: c is not defined
}

Biến toàn cục vs cục bộ: khai báo ngoài mọi hàm → toàn cục (mọi nơi truy cập được, hạn chế dùng); khai báo trong hàm/khối → cục bộ. Quy ước hiện đại: mặc định const, cần gán lại mới dùng let, tránh var.

6 Toán tử số học & gán Cơ bản

NhómToán tửVí dụ
Số học+  -  *  /  % (chia dư)  ** (luỹ thừa)7 % 2 → 1, 2 ** 10 → 1024
Tăng/giảm++  --i++ (dùng xong mới tăng), ++i (tăng rồi mới dùng)
Gán=  +=  -=  *=  /=  %=x += 5 ⇔ x = x + 5

Dấu + hai vai: gặp chuỗi là nó nối chuỗi chứ không cộng: "5" + 3 → "53", nhưng "5" - 3 → 2 (trừ thì lại ép về số!). Muốn cộng số từ ô input, hãy đổi kiểu trước bằng Number() / parseFloat() — vì input.value luôn là chuỗi.

7 So sánh == vs === Cơ bản

== ép kiểu trước khi so sánh → kết quả "bất ngờ". === so sánh cả giá trị và kiểu → dự đoán được.

Thử ngay — máy so sánh 2 giá trị
// chọn cặp giá trị rồi bấm so sánh
0 == ""          // true  (!) — "" bị ép thành 0
1 == "1"         // true  (!) — "1" bị ép thành 1
null == undefined // true  (!)

0 === ""         // false — khác kiểu là false ngay
1 === "1"        // false
5 === 5           // true  — cùng kiểu, cùng giá trị

Quy ước nghề: luôn dùng === / !==. Chỉ dùng == khi có lý do rõ ràng (gần như không bao giờ). Đây cũng là câu trắc nghiệm kinh điển: với var x = 5 thì x === "5" → false.

Toán tửÝ nghĩaVí dụ

const ketQua = diem >= 5 ? "Đậu" : "Rớt";

// && và || còn dùng để "chọn giá trị" (short-circuit):
const ten = tenNhap || "Khách";   // tenNhap rỗng/falsy → dùng "Khách"

9 Truthy & Falsy Nâng cao

Trong if, mọi giá trị đều bị quy về đúng/sai. Chỉ có đúng 6 giá trị falsy — thuộc lòng chúng là hết bẫy.

// 6 giá trị FALSY (coi như false):
false   0   ""   null   undefined   NaN

// MỌI giá trị còn lại đều TRUTHY — kể cả những "kẻ gây bất ngờ":
"0"   "false"   [ ]   { }   // đều truthy!

// Nhờ vậy viết được gọn:
if (name) { ... }        // "nếu name khác rỗng/null/undefined"
if (arr.length) { ... }  // "nếu mảng không rỗng" (0 là falsy)
Thử ngay — máy kiểm tra truthy / falsy
// thử lần lượt: 0, "0", "", [], {}, null, NaN, "false"

10 Mẹo & lỗi thường gặp Tổng kết

Lỗi 1. Quên rằng input.value là chuỗi: "5" + 3 → "53". Luôn parseFloat() / Number() trước khi tính toán.

Lỗi 2. Gán biến chưa khai báo (num = 5;) → vô tình tạo biến toàn cục. Bật 'use strict' để JS báo lỗi ngay.

Lỗi 3. Sao chép object bằng = rồi ngạc nhiên vì "sửa một cái, đổi cả hai" — xem lại mục 3 (reference).

Mẹo. Mặc định const → cần gán lại mới let → không bao giờ var. Luôn ===. Thuộc lòng 6 giá trị falsy.
