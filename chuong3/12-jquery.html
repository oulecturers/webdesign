1 jQuery là gì? Cách nhúng Cơ bản

<!-- nhúng từ CDN (đặt TRƯỚC code jQuery của mình) -->
<script src="https://code.jquery.com/jquery-3.7.1.min.js"></script>

<script>
  // cú pháp chung:  $(selector).action()
  $(function () {
    // code ở đây chạy khi DOM sẵn sàng — viết đầy đủ là $(document).ready(fn)
    // (câu trắc nghiệm 24!)
  });
</script>

Ký hiệu $ chỉ là tên hàm của jQuery. $(selector) chọn phần tử (cú pháp CSS như querySelectorAll) và trả về "đối tượng jQuery" có sẵn hàng trăm phương thức nối chuỗi được.

2 Selector & sự kiện Cơ bản

$("#id")  $(".class")  $("ul li")     // selector CSS quen thuộc
$("ul li:contains('Hot')")              // mở rộng riêng của jQuery (câu 30)

$("#btn").click(function () {           // gắn sự kiện click
  $(this).text("Đã bấm!");              // this = phần tử đang bấm (câu 27)
});

// .on() cho phần tử THÊM SAU NÀY (event delegation — câu 29):
$("body").on("click", ".btn", function () { ... });

4 Nội dung: text · html · val · attr Cơ bản

// ví dụ giáo trình: chào theo tên nhập vào
$("#btnName").click(function () {
  let name = $("#name").val();            // val(): giá trị ô input
  $("#helloId").html(name ? `Hello: <b>${name}</b>`
                           : "Please enter your name!");
});
// text() ↔ textContent · html() ↔ innerHTML · attr() ↔ get/setAttribute (câu 23)
Thử ngay — bản chạy thật của ví dụ trên

5 Thêm / xoá phần tử Cơ bản

$("#demo").append("<li>C</li>");   // thêm vào CUỐI bên trong (câu 25, 28)
$("#demo").prepend("<li>A</li>");  // thêm vào ĐẦU bên trong
$("#demo").before("<h3>…</h3>");   // chèn TRƯỚC phần tử (bên ngoài)
$("#demo").after("<h3>…</h3>");    // chèn SAU phần tử (bên ngoài)
$("#demo li:last").remove();        // xoá hẳn · empty(): xoá hết CON bên trong

6 CSS & class Cơ bản

$("p").addClass("noi-bat");       // ↔ classList.add
$("p").removeClass("noi-bat");    // ↔ classList.remove
$("p").toggleClass("noi-bat");    // ↔ classList.toggle
$("p").css("color", "red");       // ↔ e.style.color = "red"

$("#hop").outerWidth();            // rộng gồm padding + border (câu 26)
                                    // width() < innerWidth() < outerWidth()

7 Đối chiếu jQuery ↔ vanilla JS Quan trọng

Bảng "phiên dịch" để đọc code cũ rồi viết lại bằng JS thuần — kỹ năng thực tế nhất của bài này.

ViệcjQuery (cũ)Vanilla JS (mới)
Chọn 1 / nhiều$("#id") / $(".c")querySelector / querySelectorAll
DOM sẵn sàng$(document).ready(fn)DOMContentLoaded hoặc <script defer>
Sự kiện.click(fn) / .on()addEventListener("click", fn)
Nội dung.text() / .html() / .val()textContent / innerHTML / .value
Class.addClass() / .toggleClass()classList.add() / classList.toggle()
Thêm phần tử.append(html)insertAdjacentHTML("beforeend", html)
Hiệu ứng.fadeIn() / .slideUp()đổi class + CSS transition
Gọi API$.ajax() / $.get()fetch() + async/await (bài 3.12)

8 Góc nhìn thực tế & mẹo Tổng kết

Vì sao jQuery "hạ nhiệt"? Vanilla JS hiện đại đã có sẵn querySelector, classList, fetch… làm được hầu hết việc jQuery từng độc quyền; trang lớn thì dùng framework (React/Vue). jQuery còn sống trong hàng triệu trang legacy — nên vẫn phải đọc hiểu.

Mẹo thi. Các câu trắc nghiệm 23–30 đều nằm trong bài này: attr(), $(document).ready, append, outerWidth, $(this), on("click", ".btn"), :contains().

Lưu ý. Đừng trộn lẫn: $("#id").value (sai — jQuery dùng .val()) hay document.querySelector().val() (sai — DOM dùng .value). Xác định rõ đang cầm object loại nào.

Ghi nhớ. Học jQuery để đọc code cũ; bài tập & dự án mới viết vanilla JS (3.7–3.9).
