1 if / else / else if Cơ bản

Rẽ nhánh một chiều (chỉ if), hai chiều (if/else), nhiều chiều (if/else if/else).

if (diem >= 5) {
  console.log("Đậu");
} else if (diem >= 4) {
  console.log("Thi lại");
} else {
  console.log("Rớt");
}
Thử ngay — xếp loại điểm
/ 10
// nhập điểm 0–10 rồi bấm — thử 9, 6.5, 4.2, 3

Mẹo viết nhánh: xếp điều kiện từ cao xuống thấp (≥ 8, rồi ≥ 6.5, rồi ≥ 5…) thì mỗi nhánh không phải viết lại cận trên. Không nên lồng if quá ~4 cấp — tách hàm hoặc return sớm cho dễ đọc.

2 Toán tử ba ngôi ?: Cơ bản

if/else viết gọn trong một biểu thức — hợp khi chỉ chọn 1 trong 2 giá trị.

// điều_kiện ? giá_trị_khi_đúng : giá_trị_khi_sai
let ketQua = diem >= 5 ? "Đậu" : "Rớt";

// dùng thẳng trong template literal:
kq.textContent = `Bạn đã ${diem >= 5 ? "ĐẬU" : "RỚT"} kỳ thi này`;

Ba ngôi lồng nhau nhiều tầng rất khó đọc — quá 2 tầng thì quay về if/else hoặc bảng tra cứu object.

3 switch … case Cơ bản

Kiểm tra nhiều giá trị của cùng một biến — gọn hơn chuỗi if/else if. So khớp bằng === (nghiêm ngặt).

switch (thu) {
  case 7:                               // case 7 KHÔNG break →
  case 1:                               // "rơi" xuống gộp chung với case 1
    console.log("Cuối tuần"); break;
  default:
    console.log("Ngày thường");
}
Thử ngay — hôm nay thứ mấy?
// chọn thứ rồi bấm

Bẫy quên break: thiếu break → chương trình "rớt" (fall-through) chạy tiếp các case phía sau. Đôi khi cố ý gộp case (như 7 và 1 ở trên), nhưng thường thiếu break là lỗi.

4 while · do…while Cơ bản

// while: kiểm tra TRƯỚC
// → có thể không chạy lần nào
let i = 1;
while (i <= 5) {
  console.log(i);
  i++;              // quên dòng này
                    // → lặp VÔ TẬN!
}
// do…while: chạy TRƯỚC ít nhất
// 1 lần rồi mới kiểm tra
let traLoi;
do {
  traLoi = prompt("Mật khẩu?");
} while (traLoi !== "123");

Lặp vô tận là lỗi kinh điển của while: điều kiện không bao giờ sai (thường do quên tăng biến đếm) → treo tab trình duyệt. Luôn tự hỏi: "điều gì làm điều kiện này sai đi?"

5 Vòng lặp for Cơ bản

Ba phần trên một dòng: khởi tạo ; điều kiện ; thay đổi — chuẩn mực khi cần biến đếm/chỉ số.

for (let i = 1; i <= 10; i++) {
  console.log(`${bang} × ${i} = ${bang * i}`);   // template literal (bài 3.7)
}

6 for…of · for…in Nâng cao

of duyệt phần tử (mảng/chuỗi); in duyệt chỉ số/khoá (mảng/object). Một chữ khác nhau — công dụng khác hẳn.

let a = [5, 6, 3, 9];

for (let i = 0; i < a.length; i++) console.log(a[i]);  // theo CHỈ SỐ (cổ điển)
for (let x of a) console.log(x);       // theo PHẦN TỬ  → 5 6 3 9
for (let i in a) console.log(i, a[i]); // theo CHỈ SỐ/KHOÁ → "0" 5, "1" 6…

// for…in hợp với OBJECT:
const sv = { ten: "An", tuoi: 20 };
for (let khoa in sv) console.log(khoa, "=", sv[khoa]);  // ten = An, tuoi = 20

7 break & continue Cơ bản

LệnhTác dụng
breakThoát hẳn vòng lặp gần nhất (hoặc switch)
continueBỏ qua lượt này, nhảy sang lượt tiếp theo (chỉ dùng trong vòng lặp)
 
for (let i = 1; i <= 10; i++) {
  if (i === 7) break;        // gặp 7 → dừng hẳn: in 1..6
}
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) continue; // chẵn → bỏ qua: in 1 3 5 7 9
  console.log(i);
}

9 Mẹo & lỗi thường gặp Tổng kết

Lỗi 1. if (x = 5) — dấu = là gán (luôn truthy), so sánh phải là ===. Một dấu bằng thiếu, cả chương trình sai.

Lỗi 2. Quên break trong switch → fall-through; quên i++ trong while → lặp vô tận.

Lỗi 3. Điều kiện biên sai: i <= a.length chạy lố 1 phần tử (phần tử cuối có chỉ số length - 1) → undefined.

Ghi nhớ. Giá trị → for…of; chỉ số → for; khoá object → for…in; 2 lựa chọn → ba ngôi; nhiều giá trị 1 biến → switch (+ break!).
