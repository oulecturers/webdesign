1 BOM là gì? · window Cơ bản

Browser Object Model — tập đối tượng để tương tác với cửa sổ trình duyệt (nằm ngoài tài liệu HTML/DOM).

Đối tượngÝ nghĩa
windowGốc của BOM — mọi biến/hàm toàn cục đều là thành viên của nó (window.alert === alert)
locationURL hiện tại: xem, chuyển trang, tải lại
historyLịch sử duyệt: back / forward
navigatorThông tin trình duyệt: tên, ngôn ngữ, online…
screenThông tin màn hình: độ phân giải

2 location Cơ bản

location.href;      // URL đầy đủ
location.protocol;  // "https:" (hoặc "file:" khi mở file trực tiếp)
location.hostname;  // tên miền, vd "ou.edu.vn"
location.pathname;  // đường dẫn, vd "/tuyensinh/index.html"
location.port;      // cổng (Live Server thường là "5500")

location.assign("https://ou.edu.vn");  // chuyển sang trang khác
location.reload();                       // tải lại trang hiện tại
Thử ngay — mổ xẻ URL của chính trang này
// mở bằng Live Server và bằng file:// sẽ thấy protocol khác nhau

3 history Cơ bản

history.back();     // = bấm nút ← Back của trình duyệt
history.forward();  // = bấm nút → Forward
history.length;     // số trang trong lịch sử của tab này
Thử ngay — nút Back tự chế
(đi từ trang mục lục sang đây rồi bấm thử)

6 Cookie Nâng cao

Mẩu dữ liệu nhỏ lưu ở client, tự gửi kèm mỗi request lên server — sinh ra để server "nhớ mặt" người dùng (phiên đăng nhập).

// ghi (mỗi lần một cookie; kèm được Expires / Path / Secure…)
document.cookie = "theme=dark; max-age=86400; path=/";

// đọc: trả về MỘT chuỗi gộp tất cả — phải tự tách
document.cookie;   // "theme=dark; user=an"

Cookie vs Web Storage: cookie nhỏ (~4KB), tự gửi kèm mọi request (tốn băng thông), API lủng củng. Lưu dữ liệu chỉ dùng phía client (theme, giỏ hàng demo) → localStorage tiện hơn hẳn (mục 7). Cookie để dành cho thứ server cần đọc.

localStorage.setItem("theme", "dark");    // lưu (key, value — đều là CHUỖI)
let theme = localStorage.getItem("theme");  // đọc ("dark", hoặc null nếu chưa có)
localStorage.removeItem("theme");           // xoá một khoá
localStorage.clear();                        // xoá tất cả

// lưu MẢNG/OBJECT: phải đổi qua chuỗi JSON (bài 3.12 học kỹ JSON)
localStorage.setItem("todos", JSON.stringify(["học bài", "làm bt"]));
let todos = JSON.parse(localStorage.getItem("todos")) || [];

9 An toàn dữ liệu phía client CLO4

Không lưu thông tin nhạy cảm (mật khẩu, token bí mật, số thẻ) trong localStorage/cookie phía client — JS nào chạy trên trang cũng đọc được, nên một lỗ hổng XSS (bài 3.8) là bị đánh cắp sạch. Client chỉ lưu thứ mất cũng không sao: theme, giỏ hàng demo, bản nháp.

10 Mẹo & lỗi thường gặp Tổng kết

Lỗi 1. localStorage chỉ chứa chuỗi: lưu object/mảng quên JSON.stringify → đọc ra "[object Object]".

Lỗi 2. Khoá chưa từng lưu → getItem trả null — nhớ || giá_trị_mặc_định khi đọc.

Mẹo. Mẫu chuẩn cho dự án: const data = JSON.parse(localStorage.getItem("key")) || []; khi nạp trang, và một hàm luu() gọi sau mỗi lần thêm/xoá/sửa.

Ghi nhớ. location = URL · history = back/forward · navigator = thông tin máy · localStorage = nhớ bền (không gửi server) · nhạy cảm thì đừng lưu client.
