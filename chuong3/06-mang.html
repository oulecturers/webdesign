Mảng (Array)

1 Tạo & truy cập Cơ bản

let a = [3, 7, 10];             // cách thường dùng: array literal [ ]
let tapNham = [1, "hai", true]; // phần tử được phép khác kiểu

a[0];          // 3   — chỉ số bắt đầu từ 0
a[a.length-1]; // 10  — phần tử cuối
a.length;      // 3   — số phần tử
a[1] = 99;     // gán đè: [3, 99, 10]
a[9];          // undefined — vượt biên KHÔNG báo lỗi (cẩn thận!)

2 Duyệt mảng Cơ bản

let a = [3, 7, 10];

for (let i = 0; i < a.length; i++)  // ① for cổ điển — có chỉ số i
  console.log(i, a[i]);

for (let x of a)                      // ② for…of — gọn khi chỉ cần giá trị
  console.log(x);

a.forEach((value, index) => {          // ③ forEach — callback nhận (giá trị, chỉ số)
  console.log(index, value);
});

Playground — thao tác trực tiếp lên mảng
// mảng khởi đầu: ["cam", "táo", "ổi"] — bấm các nút để biến đổi

Câu trắc nghiệm kinh điển: ["green"] rồi push("blue") rồi unshift("red") → ["red", "green", "blue"] — push vào cuối, unshift chen vào đầu.

4 Tìm kiếm Cơ bản

let a = [3, 2, 4, 2, 5, 2, 7];

a.indexOf(2);       // 1  — vị trí xuất hiện ĐẦU tiên (không thấy → -1)
a.lastIndexOf(2);   // 5  — vị trí xuất hiện CUỐI cùng
a.includes(2);      // true — có tồn tại không (gọn nhất để kiểm tra)
a.find(x => x > 3); // 4  — PHẦN TỬ đầu tiên thoả điều kiện
a.findIndex(x => x > 3); // 2 — CHỈ SỐ của phần tử đó

Mẹo từ đề trắc nghiệm: phần tử xuất hiện đúng một lần khi indexOf(x) === lastIndexOf(x) — với [2,3,2,3,4,5] lọc theo điều kiện này được [4, 5].

5 every · some · filter Nâng cao

Phương thứcCâu hỏi nó trả lờiTrả về
every(fn)Tất cả phần tử đều thoả?true/false
some(fn)Tồn tại phần tử thoả?true/false
filter(fn)Lấy các phần tử thoảmảng mới
 
const diem = [8, 6.5, 9, 4, 7];

diem.every(d => d >= 5);   // false — có bạn 4 điểm
diem.some(d => d >= 9);    // true  — có ít nhất một điểm 9
diem.filter(d => d >= 5);  // [8, 6.5, 9, 7] — danh sách đậu

6 map · join · slice · concat Nâng cao

let a = [3, 2, 4];

a.map(x => x * 2);      // [6, 4, 8]  — ánh xạ từng phần tử → mảng MỚI cùng độ dài
a.join(" - ");          // "3 - 2 - 4" — nối thành CHUỖI
a.slice(1);             // [2, 4]     — cắt từ vị trí 1 → mảng mới (gốc nguyên)
a.slice(-2);            // [2, 4]     — số âm: đếm từ CUỐI (câu trắc nghiệm quen!)
a.concat([9, 8]);       // [3, 2, 4, 9, 8] — nối 2 mảng
a.reverse();            // [4, 2, 3]  — ⚠ đảo TRỰC TIẾP mảng gốc

Nhóm "hiền" vs nhóm "dữ": map/filter/slice/concat trả về mảng mới, mảng gốc nguyên vẹn; còn push/pop/splice/sort/reverse sửa trực tiếp mảng gốc. Phân biệt rõ để khỏi "mất dữ liệu oan".

7 sort — cái bẫy nổi tiếng Cơ bản

Mặc định sort() so sánh theo bảng chữ cái (chuỗi) — sắp số sẽ sai!

[10, 2, 1].sort();               // [1, 10, 2]  ✗ SAI về số! ("10" đứng trước "2")
[10, 2, 1].sort((a, b) => a - b); // [1, 2, 10]  ✓ tăng dần — truyền hàm so sánh
[10, 2, 1].sort((a, b) => b - a); // [10, 2, 1]  ✓ giảm dần

8 reduce — gộp cả mảng về một giá trị Nâng cao

let a = [3, 2, 4, 2, 5, 2, 7];

// (tong, x): tong = kết quả cộng dồn, x = phần tử hiện tại, 0 = giá trị khởi đầu
a.reduce((tong, x) => tong + x, 0);   // 25 — tính tổng

// ứng dụng thật: tổng tiền giỏ hàng
const gio = [{ ten: "Cà phê", gia: 25000 }, { ten: "Trà đào", gia: 35000 }];
const tongTien = gio.reduce((s, mon) => s + mon.gia, 0);   // 60000
Thử ngay — reduce từng bước
// nhìn "tong" lớn dần qua từng bước

9 Destructuring — trích phần tử vào biến ES6

const [first, , third] = [1, 2, 3];   // first=1, third=3 (bỏ qua phần tử giữa)
const [head, ...rest] = [1, 2, 3, 4]; // head=1, rest=[2, 3, 4]

// hoán đổi 2 biến không cần biến tạm:
let a = 1, b = 2;
[a, b] = [b, a];                       // a=2, b=1

10 Spread ... — trải mảng/object ra ES6

const a = [1, 2], b = [3, 4];

const gop    = [...a, ...b];      // [1, 2, 3, 4]  — gộp mảng
const banSao = [...a];            // [1, 2]        — bản sao nông (sửa không ảnh hưởng a)
Math.max(...[5, 9, 3]);            // 9 — trải mảng thành các đối số

const sv  = { ten: "An", tuoi: 20 };
const sv2 = { ...sv, tuoi: 21 };  // {ten:"An", tuoi:21} — copy & ghi đè 1 trường

Rest vs Spread — cùng ký hiệu ..., ngược chiều nhau: rest gom nhiều thứ thành mảng (bên trái phép gán / trong tham số hàm); spread trải mảng thành nhiều thứ (bên phải / khi gọi hàm). Đây chính là cách sao chép object an toàn đã hứa ở bài 3.2.

11 Tổng hợp: render mảng ra DOM Chuẩn nghề

Mẫu code gặp lại suốt đời làm web: mảng dữ liệu → filter → map → join → innerHTML. Xem trước kỹ thuật của bài 3.8 và dự án "Góc Cà Phê".

const menu = [
  { ten: "Cà phê sữa", gia: 25000 },
  { ten: "Bạc xỉu",    gia: 29000 },
  { ten: "Trà đào",    gia: 35000 },
  { ten: "Nước cam",   gia: 40000 },
];

ul.innerHTML = menu
  .filter(m => m.gia <= 35000)          // 1. lọc theo điều kiện
  .map(m => `<li>${m.ten} — ${m.gia}đ</li>`) // 2. mỗi object → chuỗi <li>
  .join("");                             // 3. nối lại rồi gán một lần
Thử ngay — menu "Góc Cà Phê" lọc theo giá

    12 Mẹo & lỗi thường gặp Tổng kết

    Lỗi 1. [10, 2, 1].sort() → [1, 10, 2]. Sắp số luôn truyền hàm so sánh (a, b) => a - b.

    Lỗi 2. Nhầm nhóm "hiền/dữ": tưởng sort/reverse trả mảng mới (thật ra sửa gốc), tưởng map/filter sửa gốc (thật ra tạo mới).

    Lỗi 3. So sánh mảng bằng ===: [1,2] === [1,2] → false (hai địa chỉ khác nhau — xem bài 3.2).

    Mẹo. typeof [] là "object" — kiểm tra mảng bằng Array.isArray(x). Chuỗi xử lý dữ liệu hiện đại: filter → map → join/reduce.
