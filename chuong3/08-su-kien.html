1 Sự kiện là gì? Cơ bản

Sự kiện (event) xảy ra khi người dùng (click, gõ phím, cuộn) hoặc trình duyệt (nạp xong trang, đổi kích thước) tác động đến trang. Mỗi thẻ HTML có danh sách sự kiện mà JS có thể "nghe" và xử lý.

NhómSự kiện tiêu biểu
Chuộtclick, dblclick, mousedown/up, mousemove, mouseenter/leave, contextmenu (chuột phải), wheel, drag/drop
Bàn phímkeydown (ấn), keyup (thả), keypress (cũ)
Formfocus, blur, input, change, submit
Trang / cửa sổload, DOMContentLoaded, resize, scroll

2 Hai cách gắn sự kiện Cơ bản

<!-- Cách 1: thuộc tính onxxx INLINE trong HTML — KHÔNG khuyến khích -->
<input type="button" value="Say" onclick="sayHello();" />

// Cách 2: addEventListener trong JS — KHUYẾN NGHỊ
const button = document.querySelector("#say");
button.addEventListener("click", sayHello);   // truyền TÊN hàm, không có ( )

Vì sao addEventListener thắng? (1) Tách JS khỏi HTML — dễ bảo trì; (2) gắn được nhiều handler cho cùng một sự kiện; (3) gỡ được bằng removeEventListener. Quy ước nghề: không viết onclick="" inline.

3 Đối tượng event Nâng cao

Handler nhận một tham số chứa mọi chi tiết về sự kiện vừa xảy ra.

element.addEventListener("click", (event) => {
  event.target;           // phần tử THẬT SỰ bị tác động (quan trọng nhất!)
  event.type;             // "click"
  event.clientX / clientY;// toạ độ chuột theo cửa sổ
  event.key;              // phím vừa gõ (sự kiện bàn phím)
  event.preventDefault(); // CHẶN hành vi mặc định (submit, mở link…)
});

preventDefault() dùng nhiều nhất với submit (chặn nạp lại trang để tự kiểm tra form — xem bài 14) và với phím tắt (chặn Ctrl+S mở hộp thoại lưu trang — demo mục 6).

6 Sự kiện bàn phím Nâng cao

document.addEventListener("keydown", (event) => {
  event.key;   // "a", "Enter", "ArrowUp" — KÝ TỰ/tên phím (thường dùng)
  event.code;  // "KeyA" — mã phím VẬT LÝ trên bàn phím

  if (event.ctrlKey && event.key === "s") {
    event.preventDefault();          // chặn hành vi mặc định (lưu trang)
    console.log("Bạn đã nhấn Ctrl + S!");
  }
});

8 load vs DOMContentLoaded Nâng cao

// DOMContentLoaded: HTML phân tích XONG (chưa cần chờ ảnh/CSS) — NHANH hơn
document.addEventListener("DOMContentLoaded", () => {
  // nơi an toàn để gắn sự kiện, khởi tạo trang
});

// load: nạp xong TOÀN BỘ tài nguyên (cả ảnh, video…) — muộn hơn
window.addEventListener("load", () => { ... });

Mẹo: đặt code khởi tạo trong DOMContentLoaded (hoặc dùng <script defer> / script cuối body) để chắc chắn phần tử đã tồn tại trước khi gắn sự kiện — hết lỗi "Cannot read properties of null".

9 removeEventListener Nâng cao

function chaoMotLan() { console.log("Hello!"); }

nut.addEventListener("click", chaoMotLan);      // gắn
nut.removeEventListener("click", chaoMotLan);   // gỡ — PHẢI cùng một hàm!

// ⚠ gỡ KHÔNG được nếu lúc gắn dùng hàm vô danh:
//   nut.addEventListener("click", () => {...})  → không có "tên" để gỡ
Thử ngay — nút đếm có công tắc bật/tắt listener
// gỡ xong thì nút đếm "chết" — gắn lại thì sống

10 Mẹo & lỗi: this & arrow Tổng kết

// Hàm THƯỜNG: this = phần tử nhận sự kiện ✓
button.addEventListener("click", function () {
  this.textContent = "Đã bấm!";        // this là button
});

// ARROW: this KHÔNG phải button (lấy theo phạm vi ngoài) ✗
button.addEventListener("click", () => {
  this.textContent = "...";             // ✗ this = window → lỗi!
});

// ✓ Mẹo an toàn nhất: dùng event.target — khỏi nghĩ về this
button.addEventListener("click", (e) => {
  e.target.textContent = "Đã bấm!";
});

Lỗi 1. Cần this là phần tử nhận sự kiện mà lại viết arrow function — this hoá window/undefined. An toàn nhất: dùng event.target.

Lỗi 2. addEventListener("click", chao()) — ngoặc thừa làm hàm chạy ngay khi nạp trang. Truyền tên hàm không ngoặc.

Lỗi 3. Viết "onclick" trong addEventListener — tên sự kiện là "click" (không có "on").

Ghi nhớ. Luôn addEventListener; đọc chi tiết từ event; chặn mặc định bằng preventDefault(); validate lúc blur; khởi tạo trong DOMContentLoaded.
